业务视图：左侧导航树节点层级

只比较左侧导航树怎么分层，不比较中央主区和右侧面板。

L1. 领域 → 场景
零售业务域
├─ 客户基础信息查询
├─ 客户信息变更追踪
├─ 代发明细查询
└─ 财富交易查询

公司业务域
├─ 对公账户流水查询
└─ 授信额度核查
中央主区预留

L1. 领域 → 场景

最简单，业务用户最好理解。进入树后直接看到场景，但方案资产和覆盖对象都放到中央主区再讲。

L2. 领域 → 场景组 → 场景
零售业务域
├─ 客户信息类
│ ├─ 客户基础信息查询
│ └─ 客户信息变更追踪
├─ 交易明细类
│ ├─ 代发明细查询
│ └─ 财富交易查询

公司业务域
└─ 授信核查类
   └─ 授信额度核查
中央主区预留

L2. 领域 → 场景组 → 场景

更利于规模扩张。先按业务意图分组，再进具体场景，适合后续场景数量变多的地图。

L3. 领域 → 主对象 → 场景
零售业务域
├─ 客户
│ ├─ 客户基础信息查询
│ └─ 客户信息变更追踪
├─ 代发记录
│ └─ 代发明细查询
└─ 财富产品
   └─ 财富交易查询

公司业务域
└─ 授信主体
   └─ 授信额度核查
中央主区预留

L3. 领域 → 主对象 → 场景

和对象模型更一致，利于后续从场景回到治理对象，但业务用户第一次看会稍技术一些。
