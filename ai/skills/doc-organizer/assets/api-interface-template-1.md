# API 接口规范
## 1. 文档概述

本文档定义全局的 HTTP API 规范，包括接口路径、请求/响应格式、错误码等。

## 2. 公共约定

### 2.1 请求头

| 请求头 | 是否必填 | 说明 | 示例 |
|--------|---------|------|------|
| `Authorization` | 是（鉴权接口） | JWT Token | `Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...` |
| `Content-Type` | 是（POST/PUT/PATCH） | 请求内容类型 | `application/json` |

### 2.2 统一响应结构

```json
{
  "code": "00000",
  "msg": "操作成功",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | string | 状态码，成功为 `"00000"` |
| `msg` | string | 提示信息 |
| `data` | object/array/null | 业务数据 |

### 2.3 标准状态码

| code | msg | 说明 |
|------|-----|------|
| `00000` | 操作成功 | 成功 |

### 2.4 分页约定

分页接口请求参数（统一）：

| 参数 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| `pageNum` | integer | 否 | 页码 | 1 |
| `pageSize` | integer | 否 | 每页条数 | 10 |

分页接口响应数据结构（统一）：

```json
{
  "code": "00000",
  "msg": "操作成功",
  "data": {
    "list": [],
    "total": 0
  }
}
```

| 字段 | 说明 |
|------|------|
| `list` | 数据列表 |
| `total` | 总记录数 |

### 2.5 时间格式

- 请求参数：`yyyy-MM-dd HH:mm:ss` 或时间戳
- 响应数据：`yyyy-MM-dd HH:mm:ss` 格式字符串
