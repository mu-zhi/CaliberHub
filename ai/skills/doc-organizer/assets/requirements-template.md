# 需求规格文档模板

**适用场景**：模块需求规格

## 编写指南

- **核心聚焦**：功能需求、业务规则、验收口径，不写实现细节
- 功能编号统一格式：`F-模块缩写-序号`（如 `F-MM-001` 菜单管理、`F-DM-001` 部门管理）
- 每个功能点包含：功能描述、业务规则、操作流程（如有）
- UI/交互：只写"本模块页面结构/交互规则"，通用规范引用 UI-UX 设计规范
- 性能目标：只写口径（如"列表查询 <500ms"），实现策略放后端实现
- 权限设计：写权限标识清单，接口级权限详见 API 接口文档
- 相关文档：建立与其他文档的引用关系，避免重复

**章节选择指引**：
| 章节 | 简单 CRUD | 复杂模块 |
|------|----------|---------|
| 1. 模块概述 | ✅ 必写 | ✅ 必写 |
| 2. 功能需求 | ✅ 必写 | ✅ 详细展开 |
| 3. 界面设计规范 | ⚪ 可省 | ✅ 展开 |
| 4. 权限与安全 | ✅ 必写 | ✅ 必写 |
| 5. 响应式设计 | ⚪ 引用 | ⚪ 差异点 |
| 6. 待确认事项 | ⚪ 按需 | ✅ 必写 |
| 7. 未来扩展规划 | ⚪ 不写 | ⚪ 按需 |
