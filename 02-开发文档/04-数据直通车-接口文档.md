# 数据直通车 · 接口文档

> **目标**：作为前后端联调、测试验收、版本发布的统一接口基线。  
> **范围**：覆盖口径治理阶段的导入预处理、业务场景、业务域、系统管理接口。  
> **读者**：后端工程师、前端工程师、测试工程师、产品经理、运维。  
> **当前阶段定位**：M1（口径治理 MVP）执行与稳定化阶段。

---

## 一、维护规则（强制）

1. 新增接口必须同步更新本文档。  
2. 变更接口（路径、方法、参数、返回、错误码）必须同步更新本文档。  
3. 下线接口必须记录下线日期、影响范围、替代接口。  
4. 控制器代码变更但本文档未更新，视为不满足提测条件。  
5. 本文档是接口协作唯一基线，评审和联调以本文档为准。

## 三、通用规范

### 3.1 协议与基础

- 协议：HTTP/1.1  
- 编码：`application/json; charset=UTF-8`  
- 当前实现基础路径：`/api`  
- 目标版本化路径：`/api/v1/...`（新增/重构接口优先采用）；破坏性变更使用 `/api/v2/...`，并保留至少一个兼容窗口。  
- 时间：ISO-8601（示例：`2026-02-12T15:30:00+08:00`）  
- 字段命名：`camelCase`

### 3.2 认证与权限

- 认证方式：`Authorization: Bearer <token>`。  
- 令牌获取：`POST /api/system/auth/token`。  
- 权限规则：
  - `GET /api/**`：默认可匿名访问（查询能力）。
  - `POST/PUT/DELETE /api/**`：是否需要认证由 `caliber.security.require-write-auth` 控制（当前默认 `false`）。
  - 安全总开关 `caliber.security.enabled`：当前默认 `false`（本地与联调阶段不启用登录前置）。
  - 当 `caliber.security.require-write-auth=true` 时，业务写接口（`/api/import/**`、`/api/scenes/**`、`/api/domains/**`）仅以下角色可写：`SUPPORT`、`EXPERT`、`GOVERNANCE`、`ADMIN`。
  - 当 `caliber.security.require-write-auth=true` 时，`FRONTLINE`、`COMPLIANCE` 角色默认只读，不可调用业务写接口。
  - `/api/system/**`：当 `caliber.security.require-write-auth=true` 时仅系统管理员可访问；否则默认匿名可访问。
  - `/actuator/health`、`/actuator/info`：公开。
  - `/actuator/**`（除健康检查与基础信息）：仅管理员。
- 审计字段注入：写接口中的 `operator` 字段不再作为可信来源，后端优先使用认证上下文注入操作人。
- LLM 默认运行参数（当前版本）：
  - endpoint：`https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions`
  - model：`qwen3-max`
  - apiKey：默认从 `caliber.llm.preprocess.api-key` 读取，可在系统管理页覆盖更新

### 3.3 响应规范

- 成功：直接返回业务对象（不额外包裹 `success/data`）。
- 常见状态码：

| HTTP状态码 | 说明 |
| --- | --- |
| 200 | 查询/更新/发布成功 |
| 201 | 创建成功 |
| 204 | 删除成功（无响应体） |
| 401 | 未认证或令牌无效 |
| 403 | 角色无权限 |
| 429 | 限流触发 |

### 3.4 错误响应

```json
{
  "code": "VALIDATION_ERROR",
  "message": "sceneTitle: must not be blank",
  "requestId": "7d9204a92f1747cf8f9df3f58f0f8ae9",
  "timestamp": "2026-02-12T15:30:00+08:00"
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| code | string | 错误码 |
| message | string | 错误信息 |
| requestId | string | 请求追踪 ID（对应响应头 `X-Request-Id`） |
| timestamp | string | 错误时间 |

错误相关响应头：

| Header | 说明 |
| --- | --- |
| `X-Request-Id` | 请求追踪 ID，所有接口均返回 |
| `X-Error-Code` | 错误码（仅错误响应返回） |

错误码：

| code | HTTP状态码 | 说明 |
| --- | --- | --- |
| `VALIDATION_ERROR` | 400 | 参数校验失败 |
| `DOMAIN_VALIDATION_ERROR` | 400 | 领域规则校验失败 |
| `NOT_FOUND` | 404 | 资源不存在 |
| `ILLEGAL_STATE` | 409 | 状态冲突 |
| `UNAUTHORIZED` | 401 | 未登录或令牌无效 |
| `FORBIDDEN` | 403 | 角色无权限 |
| `RATE_LIMIT_EXCEEDED` | 429 | 请求超过限流阈值 |
| `CAL-SC-409` | 409 | Scene 更新冲突（乐观锁版本不匹配） |
| `CAL-SS-409` | 409 | Service Spec 版本冲突（`spec_version` 不匹配） |
| `INTERNAL_ERROR` | 500 | 系统异常 |

### 6.2.2 `POST /api/import/preprocess-llm`

请求体：同上。  
成功响应：同上。  

行为约束：

1. 启用模型时走 LLM 抽取。  
2. 模型失败且允许回退时，改走规则抽取。  
3. 模型失败且禁止回退时，返回 `ILLEGAL_STATE`。
4. `sql_segments[].sql_raw` 必须通过后端 SQL 保真校验（去空白后 SHA-256 指纹匹配原文 SQL）；不通过时触发回退并在 `warnings` 中返回 `sql_raw_integrity_mismatch:*` 与 `sql_raw_integrity_check_failed_fallback_rule`。  
5. 当 `confidenceScore < 0.70` 时，响应标记 `lowConfidence=true`，并在 `warnings` 中返回 `confidence_below_threshold`。
5.1 低置信度场景允许预创建草稿（不丢失信息），但前端必须强提示“需人工复核”，并限制直接发布。
6. 当原文行数超过 10000 行时，直接返回 `ILLEGAL_STATE`，错误信息为“文本过长，请分段导入（当前行数：X，上限：10000）”。
7. 当请求体传 `preprocessMode=RULE_ONLY` 时，直接走规则引擎生成 `CALIBER_IMPORT_V2`，不调用大模型接口。
