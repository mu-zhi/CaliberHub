# 文档头注（必填）

	章节说明
		每份特性文档标题下方必须显式写出以下头注字段
			创建时必填
				迭代归属：说明该场景属于哪个迭代分组
				来源主文档：说明该场景回溯到哪些主方案章节
				对应实施计划：指向 `docs/plans/` 下与本场景同步产出的实施计划；若属于经批准的临时豁免，必须在此写明豁免原因与回补时机
			条件回填
				对应测试文档：工作项进入 `reviewing（评审验证中）` 前回填，指向 `docs/testing/features/` 下的对应测试文档
				状态：仅允许空值或 `done`；只在 `docs/engineering/current-delivery-status.md` 已将该工作项标记为 `done（完成）` 后回填
		若存在研究材料、口径文档或样板 `SQL（结构化查询语言，Structured Query Language）`，可追加写在来源主文档后方

# 一、特性概述

## 1.1 背景

	章节说明  
		新增功能类：出于什么样的用户需求，希望建设功能···  
		体验优化类：功能使用中，存在什么痛点，希望优化 ···  
		问题修复类：由于使用中存在什么问题，希望优化···

## 1.2 目标

	章节说明：在本特性中，我可以部分解决或者全部解决上述分析或者发现的机会，主要做的内容是什么

# 二、指标

	章节说明：完成这个场景的开发，预计几个月之后会展示效果，用什么指标证明价值，目标是多少

	编写要求
		至少同时覆盖门禁型指标与效果型指标；如果场景当前更偏平台门禁，也要明确哪些指标是“必须做到”的验收门槛
		每个关键指标都要写清统计口径，至少说明分子、分母和统计窗口；若当前阶段只能用回放集或灰度集统计，也要显式写明
		涉及拒绝、脱敏、审批、高敏控制的场景，必须把“暴露次数为 0”或等价门禁指标写成显式约束，而不是只写体验型目标

# 三、特性全景

## 3.1 特性全景描述

	章节说明：特性内的场景列表，每一个特性后续只包含一个场景

## 3.2 特性图示

	章节说明：按照实际情况提供图示，如 角色权限图、用例图、流程图、交互图、状态转换图等等

# 四、特性说明

## 4.1 原型（非必要）

章节说明：本次特性的原型截图或者链接

## 4.2 功能说明

	章节说明：按照word段落分别介绍本特性内包含的功能详细内容。参考 4.2.1 特性场景1 以此类推

	编写要求  
		每一个特性文档只写一个场景，不混写多个场景  
		除了业务说明，还要写清当前场景涉及的治理对象，如 业务场景、方案资产、输出契约、契约视图、覆盖声明、策略对象、来源契约、证据片段等  
		要明确接口或数据边界，至少说明本场景依赖哪些输入、输出哪些结果、命中哪些错误或审批边界  
		要明确前端承载页面或工作台，不只写后端处理逻辑  
		要明确本场景不负责什么，避免与相邻场景边界重叠
		每个场景都要先写“通用补充项”小结，不允许只把这些信息散写在后续条目中

### 4.2.1 特性场景1

	章节说明  
	定时任务类功能  
		可以是从外部系统同步某个数据  
		可以是定时将某个数据同步出去，或者调用外部接口  
	页面交互类功能

		覆盖范围和缺口是什么
		策略或审批边界是什么
		前端入口、详情页和跳转链路是什么
		接口或数据边界是什么；如果依赖 `BFF（前端聚合层，Backend For Frontend）`，要明确标注
		存储落位或持久化边界是什么；至少说明核心对象落在哪类存储，避免实施时反向猜测

# 五、最小完整性检查清单

1. 标题下方是否已经写出创建时必填头注“迭代归属”和“来源主文档”；若工作项已进入 `reviewing（评审验证中）`，是否已回填“对应测试文档”。
1a. 新增或重写特性文档时，是否已经同步写出“对应实施计划”；若未写计划，是否存在可追溯的批准豁免与回补时机。
2. 是否坚持一文一场景，没有把两个业务场景或两个治理场景混写在同一份文档里。
3. 指标是否同时包含门禁型指标或效果型指标，并写明分子、分母和统计窗口。
4. 是否明确治理对象，而不是只写页面行为或后端步骤。
5. 是否显式写出通用补充项：主对象、默认时间语义、覆盖范围和缺口、策略或审批边界、前端入口与跳转链路、接口或数据边界、存储落位边界。
6. 是否写清前端主承载页面，以及必要的深页承载关系。
7. 是否写清本场景的输入、输出、错误码、审批边界或降级边界。
8. 是否写清本场景不负责什么，避免与相邻特性边界重叠。
9. 若工作项已完成并进入 `done（完成）`，是否回填 `状态: done`，且与 `docs/engineering/current-delivery-status.md` 保持一致。
10. 若文档引用了主方案之外的研究材料、样板 `SQL（结构化查询语言，Structured Query Language）` 或截图材料，是否能回到具体文件。
