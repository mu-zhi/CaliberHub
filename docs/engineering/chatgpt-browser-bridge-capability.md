# ChatGPT 浏览器桥接能力

> 迭代归属：工程协作能力 / `AI（人工智能）` 协作资产
> 来源主文档：[../../AGENTS.md](../../AGENTS.md)、[./collaboration-workflow.md](./collaboration-workflow.md)、[../../ai/README.md](../../ai/README.md)

## 1. 特性概述

### 1.1 背景

当前仓库内的 `Codex（代码智能体）` / `Claude Code（代码智能体）` / 项目级 `skill（技能）` 如果想读取 `ChatGPT（对话式人工智能产品）` 已打开页面内容，或向页面输入提示，主要依赖一次性 `AppleScript（苹果脚本）` 和临时 `JavaScript（网页脚本语言）` 片段。这个方式已经证明“技术上可行”，但还没有形成稳定的工程能力：命令不统一、返回结构不稳定、失败原因不收口，也不方便后续让其他 `agent（智能体）` 直接编排调用。

本场景的核心诉求不是做通用浏览器自动化，而是把“已打开 `Google Chrome（谷歌浏览器）` 中的 `https://chatgpt.com/` 标签页读写能力”沉成一个稳定、可回归、可由其他 `skill（技能）` / `agent（智能体）` 复用的项目级能力。对上层的正式触发入口是 `skill（技能）` 与自然语言意图，不要求用户记住或手动执行底层 `CLI（命令行接口）`。

### 1.2 目标

建设一个 `ChatGPT Browser Bridge（ChatGPT 浏览器桥接器）`，由一个确定性的 `CLI（命令行接口，Command Line Interface）` 脚本和一个专用 `skill（技能）` 组成，满足以下目标：

1. 只操作“已打开”的 `https://chatgpt.com/` 标签页，不主动新开浏览器。
2. 对仓库内其他 `skill（技能）` / `agent（智能体）` 通过专用 `skill（技能）` 暴露稳定的“读取 / 输入 / 显式发送”能力；底层脚本仍保持 `list-tabs / read / type / send` 四个内部子命令，供 `skill（技能）` 编排。
3. 全部命令返回统一 `JSON（JavaScript对象表示法，JavaScript Object Notation）` 结构和退出码，便于编排与错误处理。
4. 默认安全：允许读取、允许输入，但不默认自动发送。
5. 首轮明确收敛边界，只支持 `macOS（苹果桌面系统） + Google Chrome（谷歌浏览器） + Apple 事件 JavaScript 执行已开启`。

## 2. 指标

### 2.1 门禁型指标

1. 环境前置检查命中率：在本地基线环境的 20 次回归调用中，`CLI（命令行接口）` 对“浏览器未运行、未找到目标标签页、`Apple 事件 JavaScript 执行` 未开启、页面结构缺失”四类失败原因都能返回稳定错误码和错误消息。
   - 分子：返回预期错误码且 `error.code / error.message` 完整的失败调用数
   - 分母：全部故意构造的失败调用数
   - 统计窗口：本地回归集一次完整运行
2. 默认不误发送：所有未显式触发 `send` 或 `--send-after-type` 的调用中，实际发送次数必须为 0。
   - 分子：未显式发送但实际触发发送的调用数
   - 分母：全部未显式发送的输入调用数
   - 统计窗口：本地回归集一次完整运行
3. 输出契约稳定：`list-tabs / read / type / send` 的成功输出均包含 `ok / command / url / title`，失败输出均包含 `ok=false / error.code / error.message`。
   - 分子：满足契约的命令输出数
   - 分母：全部命令输出数
   - 统计窗口：单元测试 + 本地烟测一次完整运行

### 2.2 效果型指标

1. `read` 成功率：在本地已打开 `ChatGPT（对话式人工智能产品）` 标签页的 10 次读取中，至少 9 次能返回 `messages（消息列表）` 或 `bodyText（页面正文）` 之一。
   - 分子：返回有效消息或页面正文的读取次数
   - 分母：全部读取次数
   - 统计窗口：同一基线环境下的连续烟测
2. 输入框定位成功率：在本地已打开 `ChatGPT（对话式人工智能产品）` 标签页的 10 次检测中，至少 9 次能识别 `composer.available=true`。
   - 分子：成功识别输入框可用的次数
   - 分母：全部检测次数
   - 统计窗口：同一基线环境下的连续烟测

## 4. 特性说明

### 4.1 原型（非必要）

不适用。本特性不新增项目内前端页面，交互承载直接复用已打开的 `Google Chrome（谷歌浏览器）` 标签页。

### 4.2 功能说明

#### 4.2.1 特性场景 1：已打开 ChatGPT 标签页的结构化读取与安全输入

**通用补充项**

1. 主对象：`ChatGPT Browser Bridge（ChatGPT 浏览器桥接器）` 的命令执行结果，以及被桥接的已打开 `ChatGPT（对话式人工智能产品）` 标签页。
2. 默认时间语义：不适用。本特性对“当前调用时刻的已打开标签页状态”生效，不做业务时间推理。
3. 覆盖范围和缺口：
   - 首轮覆盖：`list-tabs / read / type / send`
   - 首轮缺口：Safari / Edge / Brave、任意网站抓取、文件上传、多标签复杂会话管理、浏览器启动与登录态恢复
4. 策略或审批边界：
   - 默认不发送，只有 `send` 或 `--send-after-type` 才允许触发发送
   - `send` 优先点击发送按钮，找不到按钮时才回退到 `Enter（回车发送）`
   - 不对业务内容做自动改写；输入文本按调用参数原样写入
5. 前端入口、详情页和跳转链路：不新增仓库内前端入口；由 `skill（技能）` / `agent（智能体）` 在终端调用脚本，脚本定位现有 `Google Chrome（谷歌浏览器）` 标签页。
   - 用户侧触发方式：自然语言意图命中 `chatgpt-browser-bridge（ChatGPT 浏览器桥接技能）`
   - 内部执行方式：`skill（技能）` 调用 `scripts/chatgpt_browser_bridge.py`
6. 接口或数据边界：
   - 输入：`CLI（命令行接口）` 子命令、文本参数、输出格式参数
   - 输出：统一 `JSON（JavaScript对象表示法，JavaScript Object Notation）` 结果，成功含 `ok / command / url / title`，失败含 `ok=false / error`
   - 错误边界：浏览器未运行、目标标签页不存在、`Apple 事件 JavaScript 执行` 未开启、页面错误页、输入框不可用
7. 存储落位或持久化边界：首轮不引入持久化状态；默认只写标准输出，不落本地缓存文件，除非后续显式追加诊断开关。

**功能说明**

1. `list-tabs`：枚举当前 `Google Chrome（谷歌浏览器）` 中所有匹配 `https://chatgpt.com/` 的已打开标签页，返回 `windowIndex / tabIndex / title / url / active`，用于上游 `agent（智能体）` 做路由判断。
2. `read`：读取目标标签页中的标题、当前 URL、消息列表、页面正文摘要以及输入框可用性，返回结构化结果，供上游 `agent（智能体）` 做上下文分析。
3. `type`：向目标标签页输入框写入文本，默认只写入不发送；若显式传入 `--send-after-type`，则在写入后调用发送流程。
4. `send`：对当前输入框内容执行发送动作，优先点击按钮，再回退键盘回车。
5. 失败处理：所有子命令统一通过退出码和 `error.code` 表达失败原因，不允许仅靠标准错误的自然语言文本让上游猜测。

**本场景不负责什么**

1. 不负责通用网页自动化，不抽象成任意网站桥接层。
2. 不负责新开浏览器、登录 `ChatGPT（对话式人工智能产品）`、恢复会话或切换账号。
3. 不负责上传文件、点击复杂工具栏、选择模型或处理多模态输入。
4. 不负责把页面消息直接写入仓库真源文档；读取结果是否落仓由上游 `skill（技能）` / `agent（智能体）` 决定。
