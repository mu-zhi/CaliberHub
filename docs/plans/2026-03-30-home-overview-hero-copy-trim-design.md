# 首页总览首屏文案减重设计稿

> 设计日期：2026-03-30
> 对应正式文档：
> - `docs/architecture/frontend-workbench-design.md` §4.1
> - `docs/architecture/features/iteration-02-runtime-and-governance/12a-首页总览与状态分发.md`

## 1. 背景

当前首页 `Hero（首屏页头区）` 左侧同时存在：

1. `首页总览 / 指挥页` 轻量标识
2. 长标题“先看到当前版本、风险和待办，再进入图谱工作台”
3. 一句补充说明

这组三层文案占用了首屏垂直空间，并与下方 `一级状态卡` 的“状态优先”职责重复。

## 2. 用户确认后的边界

1. 只处理首页 `Hero` 文案减重。
2. 删除长标题和补充说明。
3. 保留 `首页总览 / 指挥页` 轻量标识。
4. 保留下方状态条。
5. 不调整动作卡、主区模块、跳转关系和整体首屏结构。

## 3. 方案对比

### 方案 A：只保留轻量标识

- 删除长标题与补充说明，只保留 `首页总览 / 指挥页`。
- 优点：最省空间，改动最小，和“状态优先”原则一致。
- 缺点：解释性文案更少，需要依赖状态卡承担首屏认知。

### 方案 B：保留短标题

- 把长标题压缩为“首页总览”等单行标题。
- 优点：仍有完整标题层级。
- 缺点：仍然占用额外一行，不是最小改动。

### 方案 C：保留一行小字说明

- 删除长标题，只保留一行说明。
- 优点：保留部分解释语义。
- 缺点：仍然占空间，且与一级状态卡职责重复。

## 4. 选定方案

采用方案 A。

原因：

1. 用户明确要求“去掉前端页面中的一些打字”，并点名该长标题“特别占空间”。
2. 首页正式设计本就强调 `一级状态卡` 承担首页第一认知，不应让说明文重新成为首屏主角。
3. 本次收口只做局部减重，不引入额外布局或信息层级调整。

## 5. 实施约束

1. 只修改 `frontend/src/pages/HomePage.jsx` 中 `home-overview-copy` 的文案结构。
2. 不改 `home-overview-statusbar` 的内容和顺序。
3. 不新增新的视觉强调元素替代被删除的文字。
4. 测试以首页渲染断言为主，验证页面在删除文案后仍可正常渲染。
