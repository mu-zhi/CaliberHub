# 设计稿与实施计划目录

本目录用于统一存放项目协作中的设计稿过程稿与实施计划，避免方案过程稿、执行清单和临时聊天记录散落在多个目录。

## 使用规则

1. `Brainstorming（方案脑暴技能）` 产出的过程稿设计文档保存到本目录。
2. `Writing Plans（实施计划编写技能）` 产出的实施计划保存到本目录。
3. 正式设计结论优先回写 `docs/` 正文；本目录只承接过程稿和实施计划，不再作为并行主文档入口。
4. 当前开发进度与下个阶段工作统一维护在 `docs/engineering/current-delivery-status.md`，不得在本目录并行维护滚动状态摘要。

## 命名约定

1. 设计稿：`YYYY-MM-DD-<topic>-design.md`
2. 实施计划：`YYYY-MM-DD-<topic>-implementation-plan.md`

## 编写要求

1. 设计稿应反映经用户确认后的当前有效方案过程稿，不保留“最终版”“修改版”等后缀。
2. 实施计划应拆成 2 至 5 分钟级别的小任务，并写清精确文件路径、完整代码、测试命令与预期输出。
3. 进入实现阶段前，应同时具备设计稿与实施计划，除非任务被明确认定为纯只读评审或纯文档同步。
