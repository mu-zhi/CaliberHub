# 审批与导出特性文档

> 迭代归属：迭代二运行与治理工作台  
> 来源主文档：`system-design.md` §6.3、§10；`frontend-workbench-design.md` §4.6、§5.2

# 一、特性概述

## 1.1 背景

本项目存在大量 `S2`、`S3`、`S4` 级敏感字段，尤其是冻结原因、名单信息、密码日志和导出诉求，不能由运行面直接放行。审批与导出若不是独立场景，高敏访问就会退化成“页面上多一个按钮”，既无法锁定版本，也无法形成审计链路。

## 1.2 目标

本特性用于把高敏访问、字段级裁剪、审批决策、脱敏导出和导出归档放在统一场景中处理，确保审批依据、审批结论、导出文件和审计记录始终挂在同一条追踪链路上。

# 二、指标

1. 所有高敏导出都必须绑定审批任务编号和追踪编号，覆盖率达到 100%。
2. 未经审批的限制字段和禁止字段暴露次数为 0。
3. 导出文件必须全部留存文件指纹、下载令牌状态和归档记录。

# 三、特性全景

## 3.1 特性全景描述

本特性只覆盖“审批与导出”一个场景。它既服务申请人，也服务审批人和合规角色，是运行面高敏决策后的正式后续链路。

## 3.2 特性图示

页面固定为待审批列表、申请详情和导出记录三层结构；审批详情页再拆成申请摘要、字段风险清单和审批建议三段。

# 四、特性说明

## 4.1 原型（非必要）

前端承载页面为“审批与导出”一级工作台及审批详情深页。

## 4.2 功能说明

### 4.2.1 特性场景1

1. 当运行面命中 `need_export（导出诉求标记）` 且涉及限制字段、高敏字段或高风险场景时，必须跳转到本场景，不允许直接下载或复制完整数据。
2. 待审批列表至少展示申请人、用途、涉及场景、风险等级、提交时间和超时状态，便于审批人先做优先级判断。
3. 申请详情必须同时展示追踪编号、业务场景、方案资产、请求字段、命中策略、字段风险清单和机器建议，审批人不能在信息不全的情况下拍板。
4. 审批动作固定为同意、拒绝、要求补充说明、改为脱敏导出四类，不允许自由扩展到无法审计的自定义动作。
5. 一旦使用 `frozen（冻结态）` 上下文进入本场景，审批依据必须锁定提交时的 `snapshot_id（快照标识）` 与 `inference_snapshot_id（推理快照标识）`，禁止因版本切换重新计算。
6. 审批通过后，由导出任务生成器创建导出任务，并记录文件指纹、脱敏方案、归档位置和下载令牌过期时间。
7. 导出记录页必须可回查审批任务编号、执行人、导出时间、字段遮蔽方案和归档状态，满足事后审计。
8. 本场景的边界是“审批与导出闭环”。它不负责重新解释场景命中逻辑，也不负责修改上游策略对象。
9. 对于 `S4` 级场景，本场景默认以拒绝或专审为主，不能用“普通审批”取代高风险专审口径。
