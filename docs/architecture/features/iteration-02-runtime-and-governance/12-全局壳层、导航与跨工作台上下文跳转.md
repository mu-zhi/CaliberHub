# 全局壳层、导航与跨工作台上下文跳转特性文档

> 迭代归属：迭代二运行与治理工作台  
> 来源主文档：`system-design.md` §16；`frontend-workbench-design.md` §2、§3.6、§4.1、§6.2
> 对应实施计划：`docs/plans/2026-04-04-global-shell-navigation-context-implementation-plan.md`

# 一、特性概述

## 1.1 背景

本项目前端不是单页工具，而是包含首页总览、数据地图、知识生产台、发布中心、运行决策台、审批与导出、监控与审计七个一级工作台的企业级工作台体系。若没有单独场景定义全局壳层、导航显隐、首页分发和跨工作台上下文包，用户在不同工作台之间切换时会丢失焦点对象、版本口径和锁定态约束。

## 1.2 目标

本特性用于统一站点级壳层、角色可见性、首页总览分发逻辑，以及 `Workbench Context Package（工作台上下文包）` 驱动的跨工作台跳转机制，确保不同页面在同一版本与同一焦点对象上连续工作。

# 二、指标

1. 一级工作台可见性一致率达到 100%。统计口径：分母为桌面端已实现的一级工作台导航位数，分子为在不同角色下仍保持“可见但按权限置灰 / 可点”的导航位数，统计窗口为首轮角色回放集。
2. 跨工作台统一上下文包使用率达到 100%。统计口径：分母为从一个工作台跳到另一个工作台的正式链路次数，分子为使用统一 `ctx（上下文包）` 承载且未散写额外临时查询参数的次数，统计窗口为当前阶段全部已实现跨台跳转。
3. `replay（回放态）` 与 `frozen（冻结态）` 跳转的版本锁定成功率达到 100%。统计口径：分母为以 `replay / frozen` 模式进入目标工作台的次数，分子为成功加载成对快照且未静默降级为 `latest（最新态）` 的次数，统计窗口为审计回放、审批详情和发布影响分析样本。

# 三、特性全景

## 3.1 特性全景描述

本特性只覆盖“全局壳层、导航与跨工作台上下文跳转”一个场景。它是所有页面共同依赖的前台底座，负责把七个工作台串成一个有状态、有锁定边界的整体系统。

## 3.2 特性图示

桌面端壳层固定为顶部横向轨道、左侧模块内导航、页头与关键状态区、主工作区四层。首页总览承担系统状态总览与任务分发，跨工作台跳转统一由上下文包驱动。

# 四、特性说明

## 4.1 原型（非必要）

前端承载页面包括首页总览与所有一级工作台的共享壳层；跨页协议由 `Workbench Context Package（工作台上下文包）` 承接。

## 4.2 功能说明

### 4.2.1 特性场景1

通用补充项：
- 主对象：站点壳层、一级导航、模块内导航、`Workbench Context Package（工作台上下文包）`、`lock_mode`、焦点对象标识、快照对。
- 默认时间语义：本场景不解释业务查询时间；默认关注导航状态、上下文包生效时点和目标页加载时点。若上下文包带快照对，则以该轮版本时点为准。
- 覆盖范围和缺口：覆盖共享壳层、角色显隐、跨工作台上下文包传递和目标页锁定态加载；不覆盖各工作台内部业务流程细节。
- 策略或审批边界：本场景不做业务审批裁定，但必须保证审批、审计和发布链路所需的 `replay / frozen` 锁定模式不被前端壳层破坏。
- 前端入口、详情页和跳转链路：承接首页总览与全部一级工作台的共享外壳；关键链路包括数据地图 -> 运行决策台、运行决策台 -> 审批与导出、发布中心 -> 影响分析、监控与审计 -> 运行决策台。
- 接口或数据边界：本场景自身只承载导航配置、上下文包解析和目标页加载协议；上下文包只传标识与模式，目标数据仍由各工作台正式接口加载。
- 存储落位边界：导航配置和角色可用性规则由前端配置层与正式接口约束共同定义，上下文包通过 URL 参数或路由状态传递，不落长期业务主数据；审计所需的跳转上下文仍由目标页的事件链留痕。

1. 顶部横向轨道固定承载品牌、七个一级工作台和全局工具区；无权限工作台保留可见但置灰，不允许直接隐藏。
2. 首页总览负责回答当前系统是否稳定、今日重点风险在哪里、我上次处理中对象是什么、我今天要处理什么、应该先进入哪个工作台五个问题，不能被弱化成普通入口页。
3. 跨工作台跳转必须统一传递上下文包，最少可携带源工作台、目标工作台、跳转意图、焦点场景、焦点方案、追踪编号、快照对、锁定模式等字段。
4. `latest（最新态）`、`replay（回放态）`、`frozen（冻结态）` 三种锁定模式必须有清晰边界：最新态允许重算，回放态用于审计复现，冻结态用于审批与发布等版本一致性场景。
5. 数据地图跳转到运行决策台、运行决策台跳转到审批与导出、发布中心跳转到影响分析、监控与审计跳转到运行决策台等关键链路，都必须在目标工作台按上下文包自动定位焦点对象，而不是让用户重新搜索。
6. 目标工作台加载失败时，必须展示结构化错误提示，说明是快照缺失、对象退役还是追踪编号失效，不允许静默降级。
7. 本场景也负责统一角色显隐、页头结构、页面模式和全局工具区位置，避免每个工作台各自发明站点壳层。
8. 本场景的边界是“保证连续工作上下文”。它不承载具体业务规则判断，也不替代单一工作台的主业务流程。
9. 所有上下文包都只传标识与模式，不传完整对象数据；目标工作台收到后必须自行取数，避免页面缓存长期漂移。
10. 首页总览的状态卡、待办和续办摘要属于 `12a` 独立场景承接；本场景只定义共享壳层、导航纪律和跨工作台上下文协议，不重复定义首页摘要内容。
11. 当 `lock_mode` 为 `replay` 或 `frozen` 时，目标工作台必须把 `snapshot_id（快照标识）` 与 `inference_snapshot_id（推理快照标识）` 缺失视为结构化错误，而不是使用默认值兜底。
12. 本场景的边界是“共享导航与跨台上下文协议”。它不替代首页摘要聚合、不承接单页数据计算，也不自行拼装目标页业务数据。
