# OpenAPI 契约与前端消费特性文档

> 迭代归属：迭代二运行与治理工作台  
> 来源主文档：`system-design.md` §9.1.3、§16；`frontend-workbench-design.md` §6.2、§7.5

# 一、特性概述

## 1.1 背景

前端当前已经涉及首页聚合、工作台跳转、数据地图、审批、运行决策和监控多个接口面。如果 `/v3/api-docs`、前端路径常量和页面消费逻辑之间没有单独特性文档承接，接口路径会重新散落回页面源码，文档与实现会再次漂移。

## 1.2 目标

本特性用于把 `OpenAPI（开放接口描述规范，OpenAPI Specification）` 生成、契约快照、前端统一消费层和跨工作台上下文包消费规则收口成一个正式场景，确保新增接口和行为变更都先经过正式契约层，再进入页面实现。

# 二、指标

1. 前端正式接口路径来自契约层的覆盖率达到 100%。统计口径：分母为页面、状态仓库和适配器中实际调用的正式后端接口数，分子为通过统一契约层路径常量或构造函数消费的接口数，统计窗口为当前已实现工作台。
2. 接口变更同步率达到 100%。统计口径：分母为影响前端交互的后端接口变更次数，分子为先更新 `OpenAPI` 输出或契约快照再修改前端实现的次数，统计窗口为当前阶段接口变更记录。
3. 首页聚合与上下文跳转契约可追溯率达到 100%。统计口径：分母为首页聚合接口和跨工作台目标页加载接口数，分子为能在 `/v3/api-docs` 中找到正式描述并在前端契约层引用的接口数，统计窗口为首轮运行与治理工作台样本。

# 三、特性全景

## 3.1 特性全景描述

本特性只覆盖“OpenAPI 契约与前端消费”一个场景。它位于运行与治理工作台的基础契约层，负责把接口设计总原则、`OpenAPI` 发布与消费规则、首页聚合接口和跨工作台上下文包加载规则收口到统一消费面。

## 3.2 特性图示

场景结构采用“左侧接口分组目录 + 中部契约快照 / 类型区 + 右侧页面消费映射区 + 底部变更校验区”的模式。中部关注接口定义，右侧关注哪些页面和工作台消费这些契约。

# 四、特性说明

## 4.1 原型（非必要）

前端主承载为统一契约层和工作台适配层，涉及首页总览、数据地图、发布中心、运行决策台、审批与导出、监控与审计等页面；跨工作台 `ctx` 加载规则与 [`12-global-shell-navigation-and-context-handoff.md`](12-global-shell-navigation-and-context-handoff.md) 配合。

## 4.2 功能说明

### 4.2.1 特性场景1

通用补充项：
- 主对象：`/v3/api-docs`、前端契约快照、路径常量 / 路径构造函数、首页聚合接口、上下文包加载接口、页面消费映射。
- 默认时间语义：本场景不承接业务查询时间；默认按接口版本、快照生成时间和前端消费版本解释当前契约状态。
- 覆盖范围和缺口：覆盖 `OpenAPI` 输出、前端统一契约层、首页聚合接口和跨工作台消费约束；不覆盖具体业务页面布局，也不替代单页业务规则文档。
- 策略或审批边界：本场景不做业务审批裁定，但任何新增接口或路径变更若未进入正式契约层，不得视为可实施。
- 前端入口、详情页和跳转链路：入口为前端契约层与各工作台适配器；重点消费链路包括首页摘要、数据地图图谱、运行决策、审批详情、监控审计和跨工作台上下文加载。
- 接口或数据边界：后端 `SpringDoc（接口文档生成框架，SpringDoc OpenAPI）` / `OpenAPI` 输出是正式真源；前端只通过统一契约层暴露路径常量、类型和请求封装，页面不得散写裸路径。
- 存储落位边界：`OpenAPI` 输出和前端生成的契约快照作为可再生产物存储在仓库内或构建产物中，运行时不维护额外业务主数据；页面调用结果仍落各自正式服务。

1. 所有会影响前端交互的后端接口都必须先进入 `/v3/api-docs`，再进入前端契约层和页面实现，不允许跳过契约快照直接在页面中拼路径。
2. 前端契约层必须统一暴露路径常量、路径构造函数、必要的请求 / 响应类型和上下文加载约束，页面、状态仓库和适配器不得散写裸路径。
3. 首页总览相关聚合接口即便由 `BFF` 提供，也必须纳入正式 `OpenAPI` 契约，不允许把首页聚合当作页面私有临时协议。
4. 跨工作台上下文包加载所需的目标页接口，也必须在契约层显式区分 `latest / replay / frozen` 三种模式，不允许目标页自行猜测默认行为。
5. 契约变更校验区必须至少说明“新增了什么接口 / 改了什么字段 / 哪些页面会受影响 / 是否需要更新上下文包加载规则”，避免后端变更只体现在 diff 中。
6. 本场景与 [12a-home-overview-and-state-dispatch.md](12a-home-overview-and-state-dispatch.md) 的边界是：`12a` 负责首页状态分发本身，本场景负责首页所消费接口的正式契约来源。
7. 本场景与 [12-global-shell-navigation-and-context-handoff.md](12-global-shell-navigation-and-context-handoff.md) 的边界是：`12` 负责上下文包协议和导航纪律，本场景负责这些协议对应的接口真源与前端消费方式。
8. 本场景的边界是“契约真源与前端消费收口”。它不负责业务场景命中，也不替代各工作台对返回结果的页面表达。
