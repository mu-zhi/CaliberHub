# 首页总览与状态分发特性文档

> 迭代归属：迭代二运行与治理工作台  
> 来源主文档：`frontend-workbench-design.md` §4.1、§6.2；`system-design.md` §9、§9.1、§9.1.3、§11、§12

# 一、特性概述

## 1.1 背景

首页总览不是普通站点欢迎页，而是七个一级工作台的统一状态入口。它要在极短时间内回答“当前系统稳不稳定、今天最重要的风险在哪里、我上次处理中断在哪、现在应该先进哪个工作台”。如果首页没有单独特性文档，后续实现极易把它简化成静态导航页，丢掉状态聚合、续办上下文和角色分发这三类核心能力。

## 1.2 目标

本特性用于把“首页总览”固化成独立前端场景，统一承接系统状态卡、重点风险、最近处理中对象、我的待办、角色提醒和跨工作台续办入口。

# 二、指标

1. 一级状态卡状态真源可追溯率达到 100%。统计口径：分母为首页四张一级状态卡的状态字段总数，分子为能回溯到正式接口、监控指标或聚合契约字段的状态字段数，统计窗口为首页主版本验收样本。
2. 首页“继续处理”上下文恢复成功率达到 100%。统计口径：分母为从首页点击“继续处理”的次数，分子为成功带着焦点对象与锁定态上下文回到原工作台的次数，统计窗口为当前阶段续办回放样本。
3. 风险区和待办区首屏可见率达到 100%。统计口径：分母为桌面端首页标准视窗下的首页渲染次数，分子为重点风险区和我的待办区均出现在首屏的次数，统计窗口为当前阶段首页布局回放样本。

# 三、特性全景

## 3.1 特性全景描述

本特性只覆盖“首页总览与状态分发”一个场景。它负责把系统状态、用户上下文和工作台分发整合到同一首页，不承担深层编辑和查询。

## 3.2 特性图示

页面结构固定为页头区、4 张一级状态卡、两列主区和辅助入口带。左列承接最近变更与最近处理中对象，右列承接重点风险、我的待办和角色提醒。

# 四、特性说明

## 4.1 原型（非必要）

前端承载页面为首页总览，对应 `frontend-workbench-design.md` §4.1。

## 4.2 功能说明

### 4.2.1 特性场景1

通用补充项：
- 主对象：一级状态卡、重点风险项、最近处理中对象、我的待办、角色提醒、首页分发入口、续办上下文包。
- 默认时间语义：首页默认按当前稳定版本时点和最近一次有效状态刷新时间组织摘要，不承接业务查询时间；续办入口若带历史锁定态，则以传入上下文时间边界为准。
- 覆盖范围和缺口：覆盖首页状态聚合、风险前置、待办摘要、续办入口和角色分发；不覆盖知识生产、发布、审批、运行决策等工作台内的完整深流程。
- 策略或审批边界：首页只表达风险和待办，不在首页直接做审批裁定或高敏字段放行；任何需要正式审批的动作必须跳转到对应工作台。
- 前端入口、详情页和跳转链路：首页为全站默认入口；四张一级状态卡、重点风险、待办和最近处理中对象都要能带上下文跳到对应工作台。
- 接口或数据边界：首页至少消费首页状态聚合接口、最近处理中对象接口、我的待办接口、重点风险摘要接口和角色提醒接口；若由 `BFF（前端聚合层，Backend For Frontend）` 承接，也必须进入正式 `OpenAPI` 契约。
- 存储落位边界：首页只消费聚合态摘要，不维护业务主数据；状态来源落各自正式服务和监控系统，首页本身只保存必要的前端缓存与当前用户视角偏好。

1. 页头区必须展示当前角色相关的一句话判断，不重复渲染站点级导航状态。
2. 一级状态卡固定为当前稳定版本、阻断发布、待审批、运行健康四张；每张卡都要可点击并带着上下文跳到对应工作台。
3. 最近处理中对象必须支持“继续处理”动作，并通过 `Workbench Context Package（工作台上下文包）` 直接把用户带回原工作台，而不是只打开一个孤立详情页。
4. 重点风险区必须优先展示发布阻断、高敏风险、元数据漂移和运行异常，不允许被普通公告替代。
5. 我的待办和角色提醒要根据角色裁剪，但一级工作台入口不按角色隐藏，只按是否可操作控制置灰。
6. 接口边界至少包括：首页状态聚合接口、最近处理中对象接口、我的待办接口、重点风险摘要接口；这些接口输出必须能回到正式追踪编号或任务编号。若由 `BFF（前端聚合层，Backend For Frontend）` 承接，也必须进入 `/v3/api-docs` 与正式版本管理，不允许页面侧私写临时聚合协议。
7. 首页只做摘要、续办和分发，不承担知识生产、运行决策、审批、发布等重操作的完整闭环。
8. 本场景的边界是“系统首屏与状态分发”。它不替代数据地图、运行决策台或监控与审计的详情页。
9. 首页上的任何状态卡和摘要项都必须能明确说明“数据来自哪里、最后更新时间是什么、点进去会进入哪个工作台”，不允许展示无法追溯来源的静态占位信息。
10. 本场景的边界是“首页摘要与分发”。它不承担二级深页逻辑，不直接编辑资产，也不在首页内重放完整审计链路。
