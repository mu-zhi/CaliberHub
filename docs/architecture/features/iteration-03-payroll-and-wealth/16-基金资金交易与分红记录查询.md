# 基金资金交易与分红记录查询特性文档

> 迭代归属：迭代三公司代发与财富样板域  
> 来源主文档：`system-design.md` §14；`frontend-workbench-design.md` §4.2、§4.5、§5.2、§5.5；研究材料：`research/source-materials/sql-samples/06-口径文档现状-基金理财保险交易查询.sql`

# 一、特性概述

## 1.1 背景

基金申请记录只能回答“客户什么时候发起了操作”，不能回答“资金什么时候真正交易、什么时候清算、什么时候分红”。业务方经常把申请日期、资金交易日期、资金清算日期和份额日期混用，因此必须把资金交易与分红记录单独成场景，否则运行时无法给出稳定时间解释。

## 1.2 目标

本特性用于把“基金资金交易与分红记录查询”沉淀为独立样板场景，按资金类日期返回申购、赎回、分红、转换等实际交易记录，并与申请记录场景明确切分。

# 二、指标

1. 资金交易场景的时间语义澄清展示覆盖率达到 100%。
2. 分红、成交金额、净值日期等资金类字段的契约兑现率达到 100%。
3. 与申请记录场景的主对象误用率控制在 5% 以下。
4. 统计口径：以上指标以基金资金类样板回放集和灰度期间请求统计；主对象误用率分母为人工标注的申请类与资金类请求数，分子为错误命中相邻场景的请求数。

# 三、特性全景

## 3.1 特性全景描述

本特性只覆盖“基金资金交易与分红记录查询”一个场景。它的主对象是基金资金事件，而非申请事件；默认关注资金交易日期和资金清算结果。

## 3.2 特性图示

运行决策台在本场景中必须前置时间语义提示卡，说明当前是资金口径；数据地图中需要突出时间语义选择器、输出契约和资金交易来源之间的关系。

# 四、特性说明

## 4.1 原型（非必要）

前端主承载为“运行决策台”，时间语义切换与覆盖说明通过内联卡片展示；覆盖差异下钻复用 `frontend-workbench-design.md` §5.5 覆盖矩阵页，审批命中时进入 §5.2 审批详情深页。

## 4.2 功能说明

### 4.2.1 特性场景1

1. 输入入口支持账户号、客户号、产品号和日期范围；若业务语句只写“基金交易”且未说明日期口径，系统默认进入本场景并提示当前按资金日期理解。
2. 输出契约应包含基金代码、产品名称、资金交易金额、资金交易日期、资金清算日期、成交份额、成交金额、成交价格、基金净值、净值日期和交易渠道等资金类字段。
3. 分红记录属于本场景，不再回到申请记录场景中混写；若用户只关心分红，可在本场景内通过交易类型继续筛分。
4. 时间语义选择器至少要区分委托日期、资金交易日期、资金清算日期和净值日期；不同日期口径带来的覆盖差异必须在结果区说明。
5. 覆盖声明应明确告知：当前阶段优先返回有可靠来源的资金事件记录；对“开户以来全量历史”查询，应在结果里提示可能存在历史缺口。
6. 策略对象对普通资金记录可以直接返回，但收益、推荐人、特殊边界字段默认按裁剪或审批处理，不把未确认收益口径直接输出。
7. 接口边界以 `Scene Search API（场景搜索接口）` 和 `Knowledge Package API（知识包接口）` 为主，知识包必须返回资金日期口径、覆盖说明、策略结果和主路径摘要，避免与申请场景共享同一输出集合。
8. 运行决策台必须提示“当前场景不返回申请时间”，若业务需要申请级信息，应引导切换到“基金申购、赎回申请记录查询”场景。
9. 本场景应与 SA 理财、受托理财、保险场景保持产品边界，不得因产品别名混淆而跨场景拼结果。
10. 本场景的边界是“基金资金事件与分红事件”。它不负责理财类交易，也不负责家族信托等专门系统场景。
