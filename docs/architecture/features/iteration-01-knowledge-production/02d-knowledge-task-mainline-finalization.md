# 知识生产台任务主线页收口特性文档

> 迭代归属：迭代一知识生产链路
> 来源主文档：`frontend-workbench-design.md` §4.3、§4.3.1
> 对应实施计划：`docs/plans/2026-04-02-knowledge-task-mainline-finalization-implementation-plan.md`

# 一、特性概述

## 1.1 背景

知识生产台任务主线页首轮重构已经把 `/production/ingest` 从“四步并列编辑页”改成“当前导入任务工作页”，但 `reviewing（测试与评审中）` 阶段仍遗留一个明确问题：`历史步骤摘要区` 虽已默认折叠且折叠态不再挂载详细内容，但摘要本身的信息密度仍偏高，导致页面首屏焦点偶尔被历史步骤而不是当前主线抢走。

## 1.2 目标

本特性只负责把“知识生产台任务主线页”收口为可稳定评审、可稳定验收的最终口径：保留当前“当前导入任务总览 + 任务主线区 + 次级结果区 + 历史步骤摘要”的工作台结构，不再讨论抽屉或标签页改造；只把历史摘要折叠态固定为四字段模板，并明确哪些内容必须删除或后置。

# 二、指标

1. 门禁型指标：`历史步骤摘要` 折叠态在 `Step 01-04` 中均只保留 4 个字段，`CTA` 数量为 `0`，多行列表数量为 `0`。
2. 门禁型指标：空态下首屏只能进入 `Step 01`，`Step 02-04` 不提供可进入动作，且该规则在前端测试与真实服务回归中都可验证。
3. 效果型指标：首屏主线区相关标题、主按钮和风险卡在浏览器级快照中保持为第一视觉，不再被历史摘要区的次级内容抢占焦点。

# 三、特性全景

## 3.1 特性全景描述

本特性只覆盖“知识生产台任务主线页收口”一个专题能力。它不新增新的工作台对象，不调整导入任务状态机，也不改知识生产链路的业务边界；它只定义现有主线页在前端的最终信息层级和折叠态字段规则。

## 3.2 特性图示

页面仍固定为四层：

1. `当前导入任务总览`
2. `任务主线区`
3. `次级结果区`
4. `历史步骤摘要`

其中只有 `历史步骤摘要` 的折叠态发生收口，不改其他三区的结构与主次关系。

# 四、特性说明

## 4.1 原型（非必要）

前端主承载页面固定为 `知识生产台 -> 材料接入与解析`，即 `/#/production/ingest` 与 `/#/production/modeling` 共享的 `KnowledgePage（知识生产页）`。

## 4.2 功能说明

### 4.2.1 特性场景1

通用补充项：
- 主对象：`Current Import Task（当前导入任务）`、`Task Mainline（任务主线）`、`History Step Summary（历史步骤摘要）`。
- 默认时间语义：不适用；本特性不定义运行时业务时间语义，只收口页面摘要字段。
- 覆盖范围和缺口：覆盖历史摘要区的折叠态字段、后置项、回跳表达和首屏信息层级；不覆盖抽屉/标签页重构，不覆盖新的导入任务状态机。
- 策略或审批边界：本特性不新增业务审批，只保留“回到上游步骤会影响后续阶段”的提示边界。
- 前端入口、详情页和跳转链路：入口固定为 `/#/production/ingest`；完成态允许通过主线 `CTA` 跳到 `/#/map/scenes`；历史摘要区不保留任何跨工作台 `CTA`。
- 接口或数据边界：沿用现有导入任务、场景队列、候选图谱和草稿队列接口；本特性不新增后端接口。
- 存储落位或持久化边界：不新增存储对象；仅收口前端展示与回归验证口径。

1. `Step 01-04` 的折叠态必须统一为 4 字段模板：`步骤标题 / 处理状态 / 指标 1 / 指标 2`。
2. `Step 01` 的两个指标固定为 `来源类型 / 导入规模（字符数）`，只回答“当前这批材料来自哪里、规模多大”；折叠态不再展示文件明细、文件大小、原文预览、上传来源说明和导入耗时。
3. `Step 02` 的两个指标固定为 `识别场景数 / 置信度`，只回答“当前识别出了多少候选场景、整体是否可信”；折叠态不再展示字段映射、逐条告警、模型供应方、模型标识、词元统计和耗时。
4. `Step 03` 的两个指标固定为 `已确认场景数 / 待处理场景数`，只回答“当前人工确认推进到哪一步”；折叠态不再展示候选实体列表、关系列表、图谱细节、`Inspector` 细节和 `graph_patch` 时间线。
5. `Step 04` 的两个指标固定为 `待发布场景数 / 当前场景状态`，只回答“发布整理是否还有剩余工作”；折叠态不再展示场景卡片预览、推荐动作、副说明和重复 `CTA`。
6. 历史摘要区折叠态不得包含链接、次级按钮、徽标组和多行日志；所有详细内容只允许后置到展开态、右侧详情区或 `/map/scenes`。
7. 本特性保留同页展开容器模型，只收口字段，不讨论抽屉或标签页；如果后续仍要调整容器模型，必须作为下一轮独立特性讨论。
8. 本特性不负责改写 `knowledge-task-mainline.js` 的任务态推导逻辑，不负责引入新的场景队列策略或跨工作台入口规则。

# 五、最小完整性检查清单

1. 是否已写明迭代归属、来源主文档和对应实施计划。
2. 是否坚持只写“知识生产台任务主线页收口”一个专题能力，没有混入真实服务 E2E、活图谱流式契约或抽屉/标签页改造。
3. 是否同时写出门禁型指标与效果型指标。
4. 是否明确 `Step 01-04` 的折叠态字段模板，而不是只写“继续压缩密度”。
5. 是否明确列出每一步必须删除或后置的内容。
6. 是否写清前端承载页面、跳转链路和不负责范围。
