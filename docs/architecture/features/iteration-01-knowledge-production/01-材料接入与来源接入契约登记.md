# 材料接入与来源接入契约登记特性文档

> 迭代归属：迭代一知识生产链路
> 来源主文档：`system-design.md` §6.1、§7；`frontend-workbench-design.md` §4.3

# 一、特性概述

## 1.1 背景

当前口径文档、历史 `SQL（结构化查询语言，Structured Query Language）`、工单样例和截图附件长期分散在不同载体中，进入系统时缺少统一登记口径。若材料进入知识生产链路前不先完成结构化登记，后续解析抽取、资产建模、元数据对齐和发布复核就会持续重复追问“这份材料到底描述什么场景、要返回什么字段、默认按什么时间解释、敏感级别是什么”。

## 1.2 目标

本特性用于把原始材料先收敛成统一的 `Source Intake Contract（来源接入契约）`，并在真实关系型控制库中形成可追溯的 `Source Material（来源材料）`、`Import Task（导入任务）` 与接入契约三类对象。系统只接受完成结构化登记、通过基础完整性校验的材料进入后续生产链路；缺失关键信息的材料必须在接入页补齐，或直接转为缺口任务。

# 二、指标

1. 来源接入契约必填项一次性填写完整率不低于 95%。
2. 高优先级材料从上传到完成登记的人工时长控制在 1 个工作日内。
3. 缺少结果字段、默认时间、覆盖范围或敏感级别的材料，阻断识别率达到 100%。
4. 首轮验收样本中的来源材料、导入任务与接入契约落库成功率达到 100%，不得只停留在接口瞬时返回体中。

# 三、特性全景

## 3.1 特性全景描述

本特性只覆盖“材料接入与登记”一个场景。它位于知识生产台的第一站，负责接收口径文档、工单样例、`SQL（结构化查询语言，Structured Query Language）` 样例和截图附件，并把材料转成后续链路唯一认可的接入契约。当前首轮以 `代发 / 薪资域` 为验收样板，但对象模型与存储边界必须具备后续复制到其他业务域的能力。

## 3.2 特性图示

本特性沿用知识生产台“顶部阶段状态条 + 主任务区 + 右侧摘要区 + 底部校验输出区”的工作台骨架。流程固定为四步：上传材料、建立来源材料与导入任务、填写来源接入契约、系统初判与人工补缺。

# 四、特性说明

## 4.1 原型（非必要）

前端承载页面为“知识生产台 -> 材料接入”子页，对应 [`frontend-workbench-design.md`](../../frontend-workbench-design.md) §4.3 中的材料接入向导；首轮验收时以 `代发 / 薪资域` 真实口径文档作为主样板。

## 4.2 功能说明

### 4.2.1 特性场景1

通用补充项：
- 主对象：`Source Material（来源材料）`、`Source Intake Contract（来源接入契约）`、`Import Task（导入任务）`。
- 默认时间语义：默认记录材料上传时间和登记时间，不在本场景内解释业务查询时间；材料中出现的业务日期仅作为待解析元信息保存。
- 覆盖范围和缺口：只覆盖“材料能否结构化进入知识生产链路”的判断；对结果字段缺失、默认时间缺失、敏感级别缺失等情况只形成阻断或缺口，不宣称已经解决后续建模问题。
- 策略或审批边界：本场景不做正式业务审批；但高敏材料必须完成敏感级别标记和材料指纹留痕，敏感级别不清时只能保存草稿。
- 前端入口、详情页和跳转链路：入口为“知识生产台 -> 材料接入”；提交成功后必须返回稳定的 `material_id（材料标识）` 与 `task_id（任务标识）`，随后进入“解析抽取”，存在阻断项时可直接跳到“缺口任务”。
- 接口或数据边界：本场景输出的是已登记材料、导入任务和来源接入契约，不直接输出正式 `Scene（业务场景）` 或 `Plan（方案资产）`；后续场景只能通过 `material_id` / `task_id` 回读本场景结果。
- 存储落位边界：来源材料、接入契约和导入任务落真实关系型控制库，原始文档、截图和附件内容落对象存储，材料指纹与基础审计事件进入事件 / 日志库。

1. 触发入口固定为“新建接入”。可上传的材料类型包括口径文档、`SQL（结构化查询语言，Structured Query Language）` 样例、工单样例和截图附件。
2. 上传成功后，系统必须先创建独立的 `Source Material（来源材料）` 记录，至少固化材料标题、来源类型、来源文件名、原文指纹、上传人、上传时间和附件引用；`Import Task（导入任务）` 只负责追踪后续状态流，不得替代来源材料主表。
3. 登记表单必须至少收集以下字段：材料标题、主对象、结果字段、默认时间语义、覆盖范围、敏感级别、来源表线索、材料出处、业务负责人。
4. 系统初判只做候选输出，不做正式治理资产落表。初判结果包括候选术语、候选业务场景、候选字段、证据片段和解析疑点，但这些结果必须通过 `material_id` / `task_id` 与来源材料形成可追溯关联。
5. 缺失项处理只有两种合法动作：当场补齐，或转缺口任务；不得带着“结果字段待确认”“时间语义待讨论”之类状态进入解析抽取。
6. 接入页右侧摘要区固定展示当前材料的关键信息摘要、完整度、敏感级别和已识别的潜在阻断项，避免用户提交后才看到问题。
7. 底部校验输出区至少提示四类错误：主对象缺失、结果字段缺失、默认时间缺失、敏感级别缺失。任一错误存在时，本次接入只能保存草稿，不能进入下一步。
8. 本特性涉及的正式治理对象只有 `Source Material（来源材料）`、`Source Intake Contract（来源接入契约）` 与 `Import Task（导入任务）` 本身，不在本场景内直接创建 `Scene（业务场景）`、`Plan（方案资产）` 或 `Source Contract（来源契约）`。
9. 审计留痕必须记录上传人、上传时间、材料类型、材料指纹、接入状态、最近一次修改人以及关联的 `task_id（任务标识）`，确保后续能追溯“这份材料是怎样进入生产链路的”。
10. 本场景的边界是“能不能进入解析抽取”。它不负责判断物理表是否可落位，也不负责判断方案资产是否可发布。
