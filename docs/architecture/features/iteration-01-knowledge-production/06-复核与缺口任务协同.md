# 复核与缺口任务协同特性文档

> 迭代归属：迭代一知识生产链路
> 来源主文档：`system-design.md` §5.4、§6.1；`frontend-workbench-design.md` §4.3、§6.1

# 一、特性概述

## 1.1 背景

即使材料已接入、证据已确认、资产已建模、来源已落位，也不意味着这些对象已经可以发布。业务、技术和合规三类复核缺一不可；同时，不完整材料和未落位字段会不断产生缺口任务，如果复核流和缺口流不协同，发布中心就会长期堆积“说不清谁卡住了什么”的阻断项。

## 1.2 目标

本特性用于建立三类复核与缺口任务的协同闭环，让每个待发布资产都能明确当前卡在哪类复核、哪条缺口、谁负责处理、何时关闭。

# 二、指标

1. 进入发布检查前，业务、技术、合规三类复核通过率必须达到 100%。
2. 所有阻断发布的缺口任务都必须具备责任人、截止时间和当前状态。
3. 复核驳回理由与缺口任务必须可追溯到具体资产和具体证据。

# 三、特性全景

## 3.1 特性全景描述

本特性只覆盖“复核与缺口任务协同”一个场景。它位于知识生产台尾段，负责承接前序场景产出的草稿资产、阻断项和缺口项，并决定哪些对象可以进入发布检查。

## 3.2 特性图示

页面固定分为左侧复核任务列表、中部复核详情区、右侧操作面板；缺口任务页固定展示任务列表、责任人、阻断等级、截止时间和状态。

# 四、特性说明

## 4.1 原型（非必要）

前端承载页面为“知识生产台 -> 复核”与“知识生产台 -> 缺口任务”两个子页。

## 4.2 功能说明

### 4.2.1 特性场景1

通用补充项：
- 主对象：复核任务、缺口任务、待发布资产与阻断项。
- 默认时间语义：本场景不承接业务查询时间；默认关注复核截止时间、缺口任务截止时间和资产版本时间。
- 覆盖范围和缺口：覆盖生产链路收口、复核通过性和缺口闭环；不覆盖正式发布执行和运行态消费解释。
- 策略或审批边界：本场景执行业务、技术、合规三类复核门禁，但不替代高敏业务审批；阻断级缺口未关闭前不得流入发布检查。
- 前端入口、详情页和跳转链路：入口为“知识生产台 -> 复核 / 缺口任务”；可从复核详情跳到缺口任务，也可从缺口任务反跳关联资产和证据。
- 存储落位边界：复核任务、缺口任务和状态机落关系型控制库，复核意见与驳回原因进入审计事件链，关联证据仍引用前序对象而不重复存储。

1. 复核任务按业务复核、技术复核、合规复核三类分组展示，并显示关联资产、责任人、截止时间、当前状态和阻断等级。
2. 中部详情区必须同时展示复核对象摘要、与上一已发布版本的差异、证据支撑、关联缺口和历史复核意见，避免复核人来回跳页拼信息。
3. 驳回不是简单状态回退，必须强制填写驳回原因，并指向具体资产字段、具体规则、具体证据或具体来源问题。
4. 缺口任务必须与资产、来源契约、输出字段或覆盖分段建立明确关联，不能创建无主任务。
5. 复核与缺口的协同规则固定为：未关闭的阻断级缺口优先处理，三类复核都通过后资产状态才能切到“已复核”。
6. 缺口任务状态至少包括打开、已分配、处理中、已解决、已豁免、已验证；其中“已豁免”只能用于明确批准的不纳入本阶段范围项，不能滥用为跳过处理。
7. 页面必须支持从复核详情直接补充缺口任务，从缺口任务反向跳回关联资产，形成闭环，而不是把两套列表割裂。
8. 本场景输出的是“哪些资产可进入发布检查，哪些资产继续阻断”，并为后续发布中心提供结构化阻断原因。
9. 本场景的边界是“生产链路收口”。它不直接执行发布、不切换快照，也不负责运行态请求解释。
