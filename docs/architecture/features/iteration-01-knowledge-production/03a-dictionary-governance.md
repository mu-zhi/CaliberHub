# 字典治理特性文档

> 迭代归属：迭代一知识生产链路  
> 来源主文档：`system-design.md` §5.1.2、§7.3；`frontend-workbench-design.md` §4.3、§5.1

# 一、特性概述

## 1.1 背景

当前口径文档、历史 `SQL（结构化查询语言，Structured Query Language）` 和样板材料中存在大量散落的码值翻译、枚举定义和业务映射。如果这些内容只保留在 `Output Contract（输出契约）` 备注、字段注释或人工经验里，同一码值会在不同场景中出现不同中文解释，后续发布、审计和运行解释都无法稳定复用。

## 1.2 目标

本特性用于把 `Dictionary（字典）` 作为一等治理资产沉淀到知识生产链路中，明确字典的建档、引用、版本、发布和差异对比规则，并与相邻的 [资产建模与治理对象编辑](03-asset-modeling-and-governance-editing.md) 场景形成稳定边界。

# 二、指标

1. 字典引用完整率达到 100%。统计口径：分母为已建模 `Output Contract` 中涉及码值翻译或枚举解释的字段数，分子为已引用正式 `Dictionary` 资产的字段数，统计窗口为首轮样板域已建模场景。
2. 同码值多口径冲突阻断率达到 100%。统计口径：分母为检测出的同一 `dict_code（字典编码）` 下重复或冲突条目数，分子为在进入已复核状态前被阻断或修正的条目数，统计窗口为知识生产链路复核样本。
3. 字典差异可追溯率达到 100%。统计口径：分母为进入发布候选包的字典变更项数，分子为能在差异视图中展示新增、修改、删除项并回溯责任人的变更项数，统计窗口为本阶段候选发布包。

# 三、特性全景

## 3.1 特性全景描述

本特性只覆盖“字典治理”一个场景。它位于知识生产链路中资产建模之后、发布之前，负责把码值翻译、枚举定义和业务映射从文本附注提升为正式治理对象，并为运行解释、输出契约兑现和审计复盘提供统一引用真源。

## 3.2 特性图示

页面采用“左侧字典列表 + 中部条目编辑区 + 右侧引用关系区 + 底部差异与校验区”的结构。主任务是建立或修正字典条目，辅助任务是确认字典被哪些契约、规则和场景引用。

# 四、特性说明

## 4.1 原型（非必要）

前端承载页面为“知识生产台 -> 资产建模”中的字典分区，深页编辑入口复用 [`frontend-workbench-design.md`](../../frontend-workbench-design.md) §5.1 资产编辑页；差异对比复用 §5.7 差异视图规范。

## 4.2 功能说明

### 4.2.1 特性场景1

通用补充项：
- 主对象：`Dictionary（字典）`、字典条目、字典版本、引用关系、差异记录。
- 默认时间语义：默认以字典版本生效时间和变更时间组织本场景，不承接业务查询时间语义。
- 覆盖范围和缺口：覆盖字典建档、条目维护、引用校验、差异对比和发布前门禁；不覆盖具体业务场景的运行命中，也不替代字段级策略裁剪。
- 策略或审批边界：本场景不做业务审批，但未引用、重复定义或冲突条目不得进入已复核或已发布状态。
- 前端入口、详情页和跳转链路：入口为“知识生产台 -> 资产建模”；可从字典详情跳到引用它的 `Output Contract`、`Derivation Rule（派生规则）` 或相邻场景。
- 接口或数据边界：本场景输入为已建模场景、契约和规则对字典的引用需求，输出为正式字典资产和引用关系；运行态只读已发布字典，不回写字典主数据。
- 存储落位边界：字典主数据、条目、版本和引用关系落关系型控制库，字典与场景 / 契约的展示关系可投影到图存储，变更记录进入审计事件链。

1. 每个 `Dictionary` 必须具备稳定的 `dict_code（字典编码）`、字典分类、版本标识和发布状态，不允许只靠标题文本区分。
2. 字典分类至少区分 `CODE_TRANSLATION（码值翻译）`、`ENUM_DEFINITION（枚举定义）` 和 `BUSINESS_MAPPING（业务映射）` 三类，不得在同一字典里混杂不同治理目的。
3. 条目编辑至少包含码值、中文解释、适用范围、来源证据、责任人和备注，不允许把核心解释埋在自由文本备注中。
4. 任何被 `Output Contract`、`Derivation Rule` 或样板场景使用的码值字段，都必须引用已建档字典，不允许直接在场景文档或 `SQL` 注释里重复写口径。
5. 右侧引用关系区必须展示“哪些场景、哪些契约、哪些规则正在使用该字典”，防止孤儿字典和双写漂移。
6. 差异视图必须按条目级别展示新增、修改、删除，并显示责任人和变更时间，不允许只展示整段文本比对。
7. 若同一码值在同一适用边界下出现两种中文解释，系统必须直接阻断并要求合并或拆分字典，不允许继续挂为候选发布。
8. 本场景与 [03-asset-modeling-and-governance-editing.md](03-asset-modeling-and-governance-editing.md) 的边界是：`03` 负责场景、方案和契约建模，本场景负责字典对象本身及其引用治理。
9. 本场景的边界是“码值和枚举口径的正式治理”。它不负责物理字段落位、不负责运行时路径选择，也不替代审批场景。
