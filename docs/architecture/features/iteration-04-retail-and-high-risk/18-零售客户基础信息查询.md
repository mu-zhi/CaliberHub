# 一、特性概述

## 1.1 背景

零售客户基础信息是很多查询场景的入口，但该场景长期依赖 `current_date - 1` 形式的快照表，容易被误理解为“任意时点都能查”。如果不先把默认快照口径固化，本项目后续关于账户、财富、变更轨迹和高风险审计的多个场景都无法稳定共用同一客户画像基线。

## 1.2 目标

本特性用于把“零售客户基础信息查询”沉淀为标准样板场景，按客户号、户口号或证件号返回当前有效的基础画像信息，并明确默认快照口径、字段级脱敏和导出审批边界。

# 二、指标

1. 当前快照口径提示覆盖率达到 100%，不允许把快照结果伪装为实时结果。统计口径：分母为命中本场景的请求数，分子为结果中显式展示快照日期与“当前快照口径”说明的请求数，统计窗口为零售客户画像样板回放集和灰度期间请求。
2. `S2（敏感级别二）` 级基础字段默认脱敏覆盖率达到 100%。统计口径：分母为返回 `S2` 级基础字段的普通查询请求数，分子为字段已按正式脱敏规则返回的请求数，统计窗口为零售客户画像样板回放集和灰度期间请求。
3. 客户号、户口号、证件号三类入口的主对象收敛准确率不低于 95%。统计口径：分母为人工标注属于客户画像场景的请求数，分子为正确收敛到客户画像主对象并命中当前场景的请求数，统计窗口为零售客户画像样板回放集和灰度期间请求。

# 三、特性全景

## 3.1 特性全景描述

本特性只覆盖“零售客户基础信息查询”一个场景。主对象是零售客户画像，主时间语义是最新可用快照日期，而不是业务发生日期。

## 3.2 特性图示

运行决策台负责展示客户画像知识包与快照提示；数据地图负责展示客户画像场景、输入槽位模式、契约视图与来源契约之间的关系。

# 四、特性说明

## 4.1 原型（非必要）

前端主承载为“运行决策台”；字段级可见范围和脱敏结果在右侧策略结果区展示；导出或高权限查看命中审批时进入 `frontend-workbench-design.md` §5.2 审批详情深页，访问与导出复盘复用 §5.3 审计追踪页。

## 4.2 功能说明

### 4.2.1 特性场景1

通用补充项：
- 主对象：零售客户画像、快照日期说明、字段级脱敏结果、输出契约、策略对象和追踪信息。
- 默认时间语义：最近可用快照日期，典型口径为 `current_date - 1` 快照；不承诺任意历史时点。
- 覆盖范围和缺口：覆盖当前有效客户画像；不覆盖账户机构变更轨迹、密码日志与高风险名单核验。
- 策略或审批边界：普通查看默认返回脱敏结果；导出或高权限查看需要审批，不允许全部角色看到同一字段集。
- 前端入口、详情页和跳转链路：主入口为运行决策台；字段可见范围与策略结果在右侧结果区展示；导出和访问复盘跳转审批详情与审计追踪页。
- 接口或数据边界：依赖 `Scene Search API（场景搜索接口）` 与 `Knowledge Package API（知识包接口）`；必须返回快照日期、字段裁剪结果和策略决策。
- 存储落位边界：场景、方案、输出契约、策略对象和字段级脱敏规则落关系型控制库；画像字段与来源契约关系投影到数据地图；访问与导出日志落审计 / 日志系统。

1. 输入入口支持客户号、户口号和证件号；系统需先做标识收敛，再进入客户画像场景，不允许直接把多入口值等价当作同一主键。
2. 默认时间语义固定为最近可用快照日期，典型实现口径为 `current_date - 1` 快照；若业务明确要求历史时点，本场景只返回当前快照说明并建议切换场景或补充需求。
3. 输出契约至少覆盖客户编号、客户姓名、证件类型、证件号码、户口号、开户信息、账户状态、归属机构和快照日期。
4. `S2` 级字段如手机号、地址、证件号等默认脱敏；普通查询不返回完整敏感值，导出或放开可见范围需要审批。
5. 覆盖声明应明确说明“当前阶段承诺当前快照画像”，不承诺任意历史时点画像回放。
6. 策略对象必须区分普通查看、导出和高权限查看三种路径，不允许所有角色看到同一字段集。
7. 接口边界以 `Scene Search API（场景搜索接口）` 和 `Knowledge Package API（知识包接口）` 为主，知识包必须带出快照日期、字段裁剪结果和策略决策，不允许把快照画像接口误当成实时画像接口。
8. 运行决策台必须显式展示“本次结果来自快照日期”，避免一线业务误以为是实时主机返回。
9. 数据地图应支持从客户画像场景下钻到相关方案资产、契约视图和来源契约，解释当前画像字段为什么能返回。
10. 本场景的边界是“当前有效客户画像”。它不负责账户机构变更轨迹、不负责密码日志，也不负责高风险名单核验。
