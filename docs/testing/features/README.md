# 测试文档目录

本目录承接项目所有特性级验收测试文档，并与 `docs/architecture/features/` 保持镜像结构。

## 映射规则

- 每份测试文档对应一份特性文档。
- 编号与中文名称保持一致，后缀固定为 `-测试报告.md`。
- 示例：
  - 特性文档：`10-数据地图浏览与覆盖追踪.md`
  - 测试文档：`10-数据地图浏览与覆盖追踪-测试报告.md`

## 特殊编号

- 如 `12a` 这类拆分场景，测试文档保留同编号，不重新整体排号。

## 归档规则

- 对应特性文档进入 `dropped` 时，测试文档移入 `_archived/` 子目录，不删除。
- 特性文档合并时，保留主文档对应测试文档，并在文中注明合并来源。

## 维护责任

- 工作项进入 `reviewing` 前，产出或更新对应测试文档。
- 测试文档的结论不能替代 `docs/engineering/current-delivery-status.md` 的状态真源。
