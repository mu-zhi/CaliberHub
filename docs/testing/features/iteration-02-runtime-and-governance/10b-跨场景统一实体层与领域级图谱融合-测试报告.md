## 一、测试范围

1. 控制库统一实体表迁移与持久化
2. 统一成员归属与统一实体关系持久化
3. 发布快照归属冻结
4. `root_type=DOMAIN` 领域级图谱聚合
5. `Impact Analysis（影响分析）` / `Runtime Retrieval（运行检索）` 复用统一身份
6. 分层越界检查、统一实体契约快照与 `Golden Path（黄金路径样例）` 烟测
7. `SCENE_MEMBERSHIP（场景成员归属）` 只能来自已发布 `scene snapshot`
8. `relation_group（关系分组）` 必须固定映射并在 `DOMAIN` 图读与投影间保持一致
9. `Formal Control（正式真源）` 与 `Snapshot（冻结可见边实例）` 不得混用

## 四、当前残余风险

1. `APPLIES_POLICY / SUPPORTED_BY` 仍未接入 `DOMAIN` 图，`relation_group` 的完整顶层回归断言目前只覆盖了 `SCENE_MEMBERSHIP / INSTANCE_OF / MAPS_TO_SOURCE`，尚未覆盖全部首轮映射。
2. `syncSourceContractsIncrementally(...)` 仍以 `scene + normalizedPhysicalTable` 复用 `Source Contract`，在“同一张表被多个方案复用”时仍可能把 `planId / notes / completeness` 等字段压到最后一个方案，`DataMapGraphDtoAdapter` 的按 `planId` 过滤路径尚未补回归验证。
3. `CanonicalEntityResolutionService` 当前只补了最小 `MAPS_TO_SOURCE` 关系，尚未加入关系删除 / 可见性去活 / `canonical_snapshot_relation_visibility` 独立冻结表，后续若进入多场景发布差异对比，需要继续扩展。
4. `Policy` 统一实体当前只覆盖 `SCENE` scope，`PLAN` scope policy 仍未进入 canonical entity / membership；若后续图谱或运行时要依赖计划级策略，需要单独补建模与测试。
