> 对应特性文档：`docs/architecture/features/iteration-02-runtime-and-governance/10-数据地图浏览与覆盖追踪.md`

## 1. 测试目标

验证“数据地图高可见层”首轮最小闭环是否满足正式门禁，重点覆盖：

1. 带 `ctx（上下文）` 进入后的自动切 `lineage（资产图谱）` 与单轮自动定位
2. 节点未命中时的明确降级提示
3. `/api/datamap/*` 作为高可见层正式主路径
4. 节点/关系详情与影响分析的最小可达性

## 2. 测试范围

本轮只覆盖“上下文进入 -> 自动图谱定位 -> 节点或关系详情 -> 影响分析 -> 跳转运行决策台”这一条最小闭环。

本轮不覆盖：

1. 多工作台并行扩展表达
2. `/api/assets/*` 旧列树能力的全面回归
3. 端到端 `E2E（端到端测试，End-to-End）` 自动化场景

## 3. 测试环境

1. 前端：`frontend`，Vite / Vitest
2. 后端：`backend`，Spring Boot / Maven
3. 当前联调口径：前端 `5173`，后端 `8080`
