## 二、验收范围与边界

### 2.1 覆盖项
- [x] 新增 `POST /api/domains/bootstrap-from-categories` 能从业务分类树初始化一级业务领域。
- [x] 初始化逻辑保持幂等，不会重复创建已存在的一级领域。
- [x] 知识生产台“重载业务领域”按钮改为先触发初始化、再刷新领域下拉。
- [x] 前端静态产物已同步到后端 `static` 目录，并在本地运行态验活。

### 2.2 不覆盖项
- 业务领域管理页的批量编辑体验优化。
- 二级、三级业务分类向场景或标签体系的进一步映射。

## 三、测试记录

- **绿测命令：** `cd backend && mvn -q -Dtest=DomainApiIntegrationTest test`
- **绿测结果：** 通过，领域初始化接口与原有领域创建/更新查询用例同时通过。

### 3.2 前端红绿验证
- **红测命令：** `cd frontend && npm test -- src/pages/knowledge-domain-actions.test.js`
- **红测结果：** 失败，缺少 `knowledge-domain-actions` 模块，符合预期。
- **绿测命令：** `cd frontend && npm test -- src/pages/knowledge-domain-actions.test.js src/pages/KnowledgePage.render.test.jsx`
- **绿测结果：** 通过，`5/5` 用例全部通过。

### 3.3 构建与静态同步
- **构建命令：** `cd frontend && npm run build`
- **结果：** 通过，仅有既有 `chunk size` 提示，无构建失败。
- **同步命令：** `bash scripts/sync_frontend_dist.sh`
- **结果：** 通过，前端产物已替换到 `backend/src/main/resources/static`。

### 3.4 服务级验活
- **后端文档探活：** `curl -s -o /dev/null -w '%{http_code}' http://127.0.0.1:8082/v3/api-docs`
- **结果：** `200`
- **前端探活：** `curl -s -o /dev/null -w '%{http_code}' http://127.0.0.1:5174/`
- **结果：** `200`
- **业务接口探活：**
  1. 登录：`POST /api/system/auth/token`
  2. 初始化：`POST /api/domains/bootstrap-from-categories`
  3. 查询：`GET /api/domains`
- **结果：** 初始化后返回 `createdCount=0, totalCount=6`，并能读取到 `零售基础业务 / 公司业务 / 贷款及信用卡业务 / 外汇及境外机构 / 财富管理 / 未分类业务领域` 六个领域，证明幂等生效。

## 四、缺陷与风险

- 当前仍保留 `未分类业务领域` 作为启动兜底；若后续确认不再需要该兜底，可再单独收口为迁移或清理任务。
- 本次只初始化一级领域；更细分类仍留在分类树与后续场景映射链路处理。

## 五、结论

- 本次修复已满足验收目标：知识生产台点击“重载业务领域”后，能够把 `01-当前业务侧的业务场景分类` 的一级分类初始化到业务领域真源，并在重复操作时保持幂等。
- 结论：**通过，可进入分支收尾或后续代码检视。**
