# 03 角色与权限边界

## 1. 文档说明

- 目标：说明不同角色“能看什么、能做什么”。  
- 范围：角色职责、模块可见性、典型协作。  
- 读者：全体角色，尤其是新接入用户。  
- 当前阶段定位：当前版本权限理解基线。  

---

## 2. 角色定义

1. 一线业务人员：重点使用首页、数据地图、个人中心，关注“找得到、看得懂”。  
2. 数据支持组：承担口径导入、修订、发布，是核心生产角色。  
3. 业务专家：参与口径校验与语义确认，关注口径准确性。  
4. 治理人员：关注资产规范性、变更影响、可追溯性。  
5. 合规人员：关注敏感信息风险、审计记录。  
6. 系统管理员：维护系统参数和大模型配置，不直接参与业务口径编辑。  

---

## 3. 模块边界

1. 业务工作区：首页、口径治理、数据地图、个人中心。  
2. 管理能力：系统管理入口仅管理员可用。  
3. 权限控制：按角色决定菜单显隐和操作可用性。  

---

## 4. 典型协作方式

1. 数据支持组导入口径并生成草稿。  
2. 业务专家复核场景定义、输入输出与 取数方案。  
3. 治理人员关注发布前门禁是否满足。  
4. 业务人员通过数据地图和场景集市消费已发布资产。  
5. 管理员按需调整 LLM 配置与系统参数。
