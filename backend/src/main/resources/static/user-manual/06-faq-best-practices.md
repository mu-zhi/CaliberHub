# 06 常见问题与最佳实践

## 1. 文档说明

- 目标：沉淀高频问题和推荐操作方式。
- 范围：导入、编辑、发布、数据地图使用。
- 读者：全体角色。
- 当前阶段定位：审计前实践沉淀。

---

## 2. 常见问题

### Q1：抽取效果不理想怎么办？

建议顺序：

1. 先看质量评分与风险提示，定位问题类型。
2. 在“抽取结果与原文对照”中快速修订关键字段。
3. 补充取数方案与码值说明。
4. 必要时直接走“手动创建草稿”路径。

### Q2：同一业务场景有多段 SQL，怎么维护？

请在取数方案管理中按“方案名 + 适用时段 + 来源表 + SQL + 备注”维护，不要把多段 SQL 粘成一个大文本块。

### Q3：服务说明和取数指南是不是一回事？

不是。
服务说明偏机器可读契约；取数指南偏业务可读说明。

### Q4：数据地图和业务场景有什么区别？

业务场景是治理对象；数据地图是消费与解释入口。
一个用于管理，一个用于浏览和追踪。

### Q5：为什么发布时比保存草稿更严格？

因为草稿阶段强调“先沉淀、再完善”；发布阶段强调“可复用、可追溯、可审计”。

---

## 3. 最佳实践

1. 导入后先修标题，再修取数方案，再补输入输出。
2. 码值说明尽量结构化维护，避免只写自然语言。
3. 风险提示用清晰短句，避免模糊描述。
4. 变更摘要要写“改了什么、为什么改、影响谁”。
5. 发布前用数据地图看一次影响范围。
6. 业务问题优先通过业务领域定位，再进入场景细看。
