## 关于知识图谱 / 数据地图方案和 NL2SQL 全链路方案的说明

当前项目将“知识图谱 / 数据地图建设”和 `NL2SQL（自然语言转 SQL，Natural Language to SQL）` 方案拆分为上下游两条方案线：

1. **知识图谱 / 数据地图方案**：解决知识供给侧问题，聚焦口径治理、数据地图、本体工程、实例图投影、共享语义资产、版本与对齐能力建设，是当前阶段的主线方案。
2. **NL2SQL 全链路方案**：解决知识消费侧问题，聚焦问句理解、检索召回、`计划中间表示（Plan Intermediate Representation）`、风险门禁、SQL 生成与执行控制，是下游消费方案。

两条方案线相互关联，但主从关系明确：

- 上游：知识图谱 / 数据地图方案提供结构化知识资产、证据链和关系语义。
- 下游：`NL2SQL` 方案消费这些资产，完成可控取数链路。

当前阶段如出现口径冲突，优先以知识图谱 / 数据地图方案为准；涉及自然语言查询、`计划中间表示（Plan Intermediate Representation）` 与 SQL 执行控制时，再以 `NL2SQL` 方案文档为准。

## 项目启动方式（本地联调）

### 1. 启动后端（Spring Boot）

```bash
cd /Users/rlc/LingChao_Ren/1.2、数据直通车/backend
mvn -q spring-boot:run
```

默认地址：`http://127.0.0.1:8080`

可用性检查（建议）：

```bash
curl -sS http://127.0.0.1:8080/api/system/llm-preprocess-config
```

说明：
1. 当前阶段默认使用 H2 内存库，适合本地开发与联调。
2. `/actuator/health` 可能因可选依赖（如 Redis）显示 `DOWN`，请以业务接口可访问性为准。
3. 大模型调用优先使用系统管理中“用户实际配置”的 API Key。

### 2. 启动前端（Vite）

```bash
cd /Users/rlc/LingChao_Ren/1.2、数据直通车/frontend
npm install
npm run dev
```

默认地址：`http://127.0.0.1:5173`

### 3. 运行规范（强制）

#### 代码修改后的服务启动原则

> **默认触发原则：每次代码修改完成后，必须自动启动前后端服务**
>
> 这是项目的核心原则之一。任何代码变更（前端或后端）完成后，必须启动/重启前后端服务，确保系统处于可访问状态。

1. **修改后必重启**：任何代码变更（前端或后端）完成后，必须启动/重启前后端服务，确保系统处于可访问状态。
2. **自动可用性校验**：服务启动后，自动执行可用性校验：
```bash
curl -sS http://127.0.0.1:8080/api/system/llm-preprocess-config
curl -sS -I http://127.0.0.1:5173
```
3. **前后端联调要求**：本地联调默认要求前后端同时在线（`8080` + `5173`），除非当前任务明确声明仅后端或仅前端。
4. **服务中断处理**：若服务中断（端口丢失、进程退出），应立即重启并在任务反馈中说明恢复时间与状态。

#### 手动启动命令

```bash
# 启动后端（Spring Boot）
cd /Users/rlc/LingChao_Ren/1.2、数据直通车/backend
mvn -q spring-boot:run

# 启动前端（Vite）
cd /Users/rlc/LingChao_Ren/1.2、数据直通车/frontend
npm run dev
```

### 4. 常见问题

1. 端口被占用（`8080` 或 `5173`）：
```bash
lsof -nP -iTCP:8080 -sTCP:LISTEN
lsof -nP -iTCP:5173 -sTCP:LISTEN
```
2. 前端已启动但页面空白：确认后端接口可访问，再刷新前端页面。
3. 导入接口耗时较长：大文档会分块处理；若 LLM 不可用会重试后回退规则抽取，耗时会增加。

### 5. 系统化测试流程（推荐）

```bash
bash scripts/run_system_test_flow.sh
```

该脚本会顺序执行：
1. 前端 `lint` / `test` / `build`
2. 前端静态产物同步到后端
3. 后端 `mvn -q test`
4. 前后端轻量可达性检查（如服务已在线）

NFR 验收门禁（P95 / 失败率 / 降级可观测）：

```bash
bash scripts/run_nfr_acceptance_gate.sh
```
