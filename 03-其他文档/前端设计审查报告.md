## 2. 本轮核心改造

### 2.1 路由与能力状态

1. 在路由元数据中新增：`implemented`、`statusLabel`、`fallbackGoal`、`fallbackPhase`、`fallbackWorkaround`。
2. 数据地图子路由按“已实现 / 后续建设”分层管理。
3. 侧栏对未实现能力显示 `[后续建设]` 标签，保证能力边界可见。

### 2.2 数据地图页面

1. `AssetsPage` 改为“同页多视图 + 状态标注”。
2. 未实现子路由统一渲染三段式占位：功能目标、预计阶段、当前替代方式。
3. 补充替代路径按钮，避免用户进入死路由。

### 2.3 可访问性与可用性

1. 核心表单补齐 `id/htmlFor/name/autocomplete`。
2. 异步反馈与错误提示增加 `aria-live="polite"`。
3. 顶层布局新增“跳转到主内容”快捷链接（Skip Link）。
4. 图谱节点支持键盘选择（Enter/Space），不再仅依赖鼠标点击。

### 2.4 文案与动效

1. 加载态文案统一使用 `…`。
2. 时间渲染统一改为 `Intl.DateTimeFormat`。
3. 增加 `prefers-reduced-motion` 降级规则。
4. 输入焦点策略改为 `:focus-visible`，避免鼠标点击误触焦点描边。

### 2.5 深链能力

1. 数据地图状态与 URL 查询参数同步：`keyword`、`domainId`、`status`、`topicId`、`mode`、`selectedId`。
2. 支持刷新、回退、分享链接后恢复筛选与模式状态。

### 2.6 Legacy 清理

1. 删除 `frontend/src/legacy/appRuntime.js`。
2. 删除 `frontend/src/legacy/legacy-body.html`。
3. 删除失效测试 `frontend/nav-click.spec.js`。

### 2.7 视觉系统升级与品牌图标

1. 全站样式切换为“现代数据平台”方向，统一采用深海蓝+青绿视觉变量，并保留现有中文信息架构。
2. 顶部壳层、侧栏导航、卡片、表单、表格、状态提示、图谱画布完成统一配色与层次重绘，降低旧纸质装饰对任务操作的干扰。
3. 新增页头品牌图标组件 `BrandMark`，并同步重绘前后端 `favicon.svg`，确保浏览器页签与站内品牌识别一致。

---

## 3. 审查结论

1. 前端主流程已满足“业务路由语义可解释 + 未实现能力可感知 + 可替代路径可执行”。
2. 可访问性基线从“局部可用”提升为“核心流程可键盘操作、状态可播报”。
3. 本轮视觉改造后，可访问性能力保持有效：Skip Link、焦点可视、图谱节点键盘可达未回退。
4. 本轮视觉改造后，`[后续建设]` 状态标注保留且与主功能态保持可辨识区分。
5. 本轮视觉改造后，图谱模式（浏览/路径/影响）的节点与关系编码仍具可区分性。
6. 仍需在后续版本继续扩展：
   - 对全部图谱交互增加更细粒度的无障碍说明。
   - 持续清理历史样式冗余，降低维护复杂度。

---

## 4. 后续评审检查单

4. 新增状态性 UI 是否支持 URL 深链。
