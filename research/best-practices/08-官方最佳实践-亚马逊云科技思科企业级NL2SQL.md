# 官方最佳实践-亚马逊云科技思科企业级NL2SQL

本文整理亚马逊云科技与思科关于企业级 `NL2SQL（自然语言转 SQL，Natural Language to SQL）` 的官方实践文章，只保留与数据直通车方案直接相关的部分。

## 1. 官方来源

1. 来源：[亚马逊云科技思科企业级 `NL2SQL（自然语言转 SQL，Natural Language to SQL）` 官方文章](https://aws.amazon.com/blogs/machine-learning/enterprise-grade-natural-language-to-sql-generation-using-llms-balancing-accuracy-latency-and-scale/)
2. 来源方：亚马逊云科技官方博客
3. 访问日期：2026-03-12

## 2. 文章最核心的做法

这篇文章最重要的不是“怎么调一个更强的模型”，而是先把企业级问题拆小。

文章明确提出两层关键动作：

1. 先把用户问题映射到具体 `Domain（业务领域）`。
2. 再基于这个领域构造领域专属上下文，让模型只处理这一小块问题。

原文里讲得很直白：企业数据天然跨多个业务域，表多、结构复杂、数据库类型不一致。想要提高生成准确率，核心动作不是把更多表塞给模型，而是把生成范围先缩到正确领域。

## 3. 官方文中值得我们直接吸收的点

### 3.1 先做领域收缩

文章明确强调，企业级 `NL2SQL（自然语言转 SQL，Natural Language to SQL）` 必须先收缩到特定数据领域。这与我们现在坚持的 `Domain（业务领域） -> Scene（业务场景）` 路由高度一致。

对数据直通车来说，这意味着：

1. 不能让下游 `NL2SQL（自然语言转 SQL，Natural Language to SQL）` 直接面对全库。
2. 上游必须先判断问题命中哪个领域，再判断命中哪个场景。

### 3.2 领域上下文不是只有表结构

文章建议把领域上下文组织成一套完整内容，而不是只有建表定义。

原文明确列到了这些元素：

1. 领域专属规则
2. 领域专属少样本问题和 `SQL（结构化查询语言，Structured Query Language）`
3. 领域专属系统提示词
4. 领域专属用户提示词
5. 领域专属临时结构或视图脚本

这对数据直通车的启发非常直接：我们上游返回的东西不能只有“表名、字段名、`JOIN（关联）` 条件”，还要有规则、边界和示例。

### 3.3 显式补充规则和 `JOIN（关联）` 提示

文章提到要用元数据增强领域建表定义，包括表、字段和规则说明，也提到领域专属关联提示。

这说明业内最佳实践并不相信模型会自己猜出复杂关联路径，而是默认要把这些信息显式提供给模型。

### 3.4 数据源解耦

文章还强调：领域上下文里要提前记录数据库方言和连接信息，这样自然语言问题就不会和具体数据源强耦合。

这部分对数据直通车的意义是：

1. 上游知识底座负责统一表达。
2. 下游执行层再决定落到哪类数据库和哪种方言。

## 4. 这篇文章和我们的方案对应在哪里

对应关系可以直接压成下面这张表：

| 官方做法 | 数据直通车中的对应物 |
| --- | --- |
| 数据领域 | `Domain（业务领域）` |
| 领域内常用结构组合 | `Scene（业务场景）` 主路径 |
| 领域专属规则 | `Derivation Rule（派生规则，Derivation Rule）`、边界规则 |
| 领域专属关联提示 | 表间关联关系对象 |
| 领域专属少样本示例 | 场景下挂载的示例 `SQL（结构化查询语言，Structured Query Language）` |
| 动态提示词组装 | 下游 `NL2SQL（自然语言转 SQL，Natural Language to SQL）` 消费问题相关业务子图 |

## 5. 我们不直接照搬的地方

这篇文章对我们很有参考价值，但不能直接照搬成最终方案，原因有 3 点：

1. 它更偏执行链路，对证据、版本、影响分析讲得不够重。
2. 它把“领域上下文”组织得很强，但没有把知识图谱治理对象展开到场景、证据、版本这一层。
3. 它默认重点是生成准确率，而数据直通车还要额外满足解释、审计和口径治理。

## 6. 当前最值得吸收的结论

如果只保留一句话，这篇文章对数据直通车最大的启发就是：

企业级 `NL2SQL（自然语言转 SQL，Natural Language to SQL）` 的关键不是让模型更懂 `SQL（结构化查询语言，Structured Query Language）`，而是先把问题缩到正确领域和正确上下文。
