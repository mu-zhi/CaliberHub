# 测试与质量文档模板

**适用场景**：模块测试用例

## 编写指南

- 用例需覆盖正常流程、边界条件、异常分支
- 用例 ID 命名规范：`T-模块缩写-序号`（如 `T-MM-001`）
- 场景分类：`正常流程：`、`边界：`、`异常：`、`验证：`
- 优先级定义：P0（核心功能）、P1（重要边界）、P2（低频场景）
- 引用而非重复：接口参数详见 API 接口文档，性能目标详见需求规格
- 测试数据使用统一前缀便于清理

**章节选择指引**：
| 章节 | 简单模块 | 复杂模块 |
|------|---------|---------|
| 1-4, 6, 8 | ✅ 必写 | ✅ 必写 |
| 5. 接口测试要点 | ⚪ 可省 | ✅ 展开 |
| 7. 性能与并发 | ⚪ 按需 | ✅ 必写 |
| 9. 数据一致性 | ⚪ 按需 | ✅ 展开 |
| 10. UI 测试要点 | ⚪ 按需 | ✅ 展开 |
| 11. 数据权限测试 | ⚪ 不写 | ✅ 按需 |
| 12. 验收标准 | ⚪ 可省 | ✅ 必写 |
| 13. 附录 | ⚪ 不写 | ⚪ 按需 |
