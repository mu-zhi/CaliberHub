# API 接口文档模板

**适用场景**：模块级别的API 接口规范。（作为该模块 API 契约的唯一权威来源）

## 编写要点

- **唯一权威性**：本文档是该模块 API 规范的唯一权威来源，其他文档只能引用
- **完整性**：每个接口必须包含：路径、方法、请求/响应、业务规则、权限标识
- **一致性**：接口命名遵循 RESTful 规范，统一使用 kebab-case
- **可维护性**：接口变更时同步更新本文档，并通知相关方（前端/测试）
- **不撰写接口详细设计**：由于已经存在API文档 MCP，所以接口文档不再重复细节，通过MCP可以查询得到。
- 仅撰写接口清单、权限标识、基础路径
- 不撰写请求参数/响应结构、Schema 定义，可通过 MCP 获取

| `/api/v1/[模块]/_import` | POST | 导入 | - |
| `/api/v1/[模块]/template` | GET | 下载导入模板 | - |
