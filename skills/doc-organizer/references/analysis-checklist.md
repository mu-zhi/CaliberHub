# 文档分析检查清单

本文档提供分层检查清单，根据任务复杂度选择对应级别。

---

## 一、快速检查（轻量流程）

> 适用场景：编写/补全/修复单个文档

### 1.1 文档规范检查（5 项）

- [ ] 文档类型与模板匹配
- [ ] SSOT 内容没有重复（API/交互/性能阈值只写一处）
- [ ] 引用链接正确（相关文档路径有效）
- [ ] 章节选择合理（简单模块省略可选章节）
- [ ] 编号规范统一（`F-XX-001`、`T-XX-001`）

### 1.2 内容边界检查（4 项）

- [ ] 需求规格：只写功能需求和交互规则，不写实现细节
- [ ] 后端实现：不粘贴 DDL，只引用数据库设计
- [ ] 前端实现：不复述需求中的交互规范
- [ ] 测试用例：只写测试点/步骤/断言，引用需求章节

---

## 二、标准检查（中量流程）

> 适用场景：文档整理/去重/迁移

### 2.1 结构问题检查

| 检查项 | 问题指标 |
|--------|---------|
| 目录深度 | > 3 层 |
| 空目录 | 存在 |
| 文件分散 | 同模块文档在多个位置 |
| 命名不一致 | 风格混杂 |

### 2.2 重复内容检查

| 检查项 | 问题指标 |
|--------|---------|
| 相同功能描述 | 出现在多个文档 |
| 复制粘贴内容 | 存在 |
| 相同图片 | 在不同位置重复存储 |
| 多语言重复 | Java/Go/Python 重复描述同一规则 |

### 2.3 引用关系检查

| 检查项 | 问题指标 |
|--------|---------|
| 内部链接 | 有死链接 |
| 图片引用 | 路径不正确或不可访问 |
| 循环引用 | 存在 |

### 2.4 SSOT 冲突检查

| 检查项 | 问题指标 |
|--------|---------|
| API 接口 | 测试/后端/前端各写一套 |
| UI 交互 | 需求/前端/测试重复描述 |
| 性能阈值 | 多处定义不同值 |
| 数据库结构 | 模块文档粘贴 DDL |
