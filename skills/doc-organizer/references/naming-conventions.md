# 文档命名规范

## 一、基本原则

| 原则 | 说明 |
|------|------|
| **清晰性** | 文件名应清晰反映文档内容 |
| **一致性** | 同一项目使用统一的命名风格 |
| **简洁性** | 避免过长的文件名（≤ 50 字符） |
| **可读性** | 使用分隔符提高可读性 |

---

## 二、命名风格

### 2.1 推荐风格

| 风格 | 格式 | 示例 | 适用场景 |
|------|------|------|---------|
| **数字前缀 + 中文** | `XX-名称.md` | `01-产品概述.md` | 需要固定排序的一级文档 |
| **纯中文** | `名称.md` | `需求规格.md` | 模块内部文件 |
| **kebab-case** | `word-word.md` | `api-interface.md` | 英文项目 |

### 2.2 数字前缀规则

```
00-项目概览.md      # 总览/导航
01-需求分析.md      # 第一阶段
02-架构设计.md      # 第二阶段
03-开发指南.md      # 第三阶段
...
```

- 使用两位数字（01, 02, 03），便于排序
- 预留数字空间用于扩展
- 保持连续性，避免跳号

---

## 三、标准文件名

### 3.1 模块内部文件（固定命名）

| 文件名 | 说明 |
|--------|------|
| `需求规格.md` | 模块功能需求、交互规则 |
| `API接口.md` | 模块接口契约 |
| `后端实现.md` | 后端业务逻辑 |
| `前端实现.md` | 前端组件设计 |
| `客户端实现.md` | Flutter/Android/iOS 实现 |
| `测试用例.md` | 测试点、步骤、断言 |

### 3.2 特殊文件命名

| 类型 | 命名方式 | 示例 |
|------|---------|------|
| 概览文档 | `00-` 前缀 | `00-项目简介.md` |
| 索引文档 | `README.md` 或 `总览.md` | `README.md` |
| 临时文档 | 特殊前缀标记 | `草稿-文档名.md`、`WIP-文档名.md` |

---

## 四、禁止的命名方式

| 类型 | ❌ 错误示例 | ✅ 正确示例 |
|------|-----------|-----------|
| 特殊字符 | `用户管理@v1.0.md` | `用户管理.md` |
| 空格 | `用户管理 模块.md` | `用户管理模块.md` |
| 过长文件名 | `用户管理模块的详细需求分析和设计方案-v1.0.0.md` | `用户管理-需求分析.md` |
| 斜杠 | `API文档/REST.md` | `API文档-REST.md` |

---

## 五、命名检查清单

- [ ] 文件名清晰反映文档内容
- [ ] 使用统一的命名风格
- [ ] 避免使用特殊字符和空格
- [ ] 文件名长度 ≤ 50 字符
- [ ] 使用数字前缀（如需要排序）
- [ ] 模块内部文件使用标准命名
