# 01 系统定位与愿景

## 1. 文档说明

- 目标：统一“我们为什么做数据直通车”的认知。  
- 范围：系统定位、解决的问题、近期目标、远期方向。  
- 读者：全体角色。  
- 当前阶段定位：面向当前版本的对齐文档。  

---

## 2. 数据直通车的定位

数据直通车是数据资产治理与消费的统一工作台。  
它不是单纯的“SQL 工具”，而是把分散的口径文档沉淀为可复用、可追溯、可发布的数据资产。

当前系统以“口径治理”作为起点，通过“导入、复核、编辑、发布”形成最小闭环，再通过“数据地图”提升资产可发现性和可理解性。

---

## 3. 我们要解决的核心问题

1. 口径分散：同类问题在不同文档重复解释，难复用。  
2. 路径迷失：业务人员不知道从哪个入口拿数据。  
3. 变更不可见：口径调整后影响范围不清晰。  
4. 治理与消费割裂：治理产出没有自然转化成可浏览的数据地图。  

---

## 4. 近期目标（当前版本）

1. 支持将口径文档导入为系统草稿。  
2. 支持结构化抽取质量判断与人工修订。  
3. 支持场景发布与基础检索。  
4. 支持以主题节点为入口浏览数据地图图谱。  

---

## 5. 愿景（持续演进）

1. 让业务用户用业务概念快速找到可用资产。  
2. 让治理用户清晰掌握资产关系、变更影响和发布质量。  
3. 让系统形成“知识生产 → 资产治理 → 数据消费”的闭环。
