# 05 质量门禁与发布说明

## 1. 文档说明

- 目标：解释系统中的质量判断与发布门禁逻辑。  
- 范围：导入质量、编辑校验、发布条件、常见拦截原因。  
- 读者：数据支持组、业务专家、治理人员。  
- 当前阶段定位：当前版本门禁说明。  

---

## 2. 质量门禁总览

系统门禁遵循“草稿可保存、发布要校验”的原则。

1. 导入阶段：允许抽取结果不完美，先生成草稿。  
2. 编辑阶段：允许反复保存草稿，不阻断修订。  
3. 发布阶段：必须满足发布必填和基础结构要求。  

---

## 3. 导入质量判断维度

1. 结构完整性：关键字段是否存在。  
2. SQL 保真率：抽取 SQL 是否可回溯到原文。  
3. 关键字段完整度：标题、输入、输出、取数方案、风险说明是否齐备。  
4. 场景拆分合理性：抽取场景粒度是否合理。  
5. 告警密度：告警与错误数量。  

说明：评分用于指导人工修订，不直接替代人工判断。

---

## 4. 发布前必填项

1. 验证时间。  
2. 变更摘要。  
3. 场景标题。  
4. 至少一条 取数方案。  
5. 输入参数与输出字段建议完整维护。  

---

## 5. 常见拦截原因与处理建议

1. 未填写验证时间或变更摘要。  
处理建议：补齐后重新发布。  

2. 取数方案为空或内容不完整。  
处理建议：至少维护一个完整方案，包含 SQL 原文。  

3. 关键字段缺失导致高风险。  
处理建议：先保存草稿，补齐字段后再发布。  

4. 码值说明缺失导致解释不足。  
处理建议：在码值说明中补充“字段 | 含义 | 映射”。  

---

## 6. 发布后建议动作

1. 在数据地图中确认关系是否符合预期。  
2. 在场景集市中验证可检索性。  
3. 如涉及关键口径变更，补充质量备注并通知相关角色。
